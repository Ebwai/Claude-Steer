# MRD 模板 (Market Requirements Document)

## 核心定位

MRD 回答「**为什么值得做？（Why）**」，聚焦**产品机会**，不讨论具体功能。

---

## 章节结构

### 1. Persona 和 Problem Statement

对每个目标用户群体，按以下格式描述：

> 【某类用户】，在【某个场景下】，我想要【某个能力】，以便【达成某个目的】

同时分析每个需求的**本质是什么**——即这个需求背后真正要解决的底层问题是什么。

示例格式：

```markdown
### Persona 1: [用户类型名称]

- **用户画像**：[描述该类用户的特征、背景、技术能力等]
- **场景**：[用户所处的具体场景]
- **核心诉求**：我想要[某个能力]，以便[达成某个目的]
- **需求本质**：[这个需求背后真正要解决的底层问题是什么]
```

### 2. 产品机会

基于 Persona 和 Problem Statement，分析：

- **市场机会**：这个产品填补了什么空白？为什么现有方案不够好？
- **价值主张**：产品能为用户创造什么核心价值？
- **成功指标**：如何衡量产品是否成功？（定性 + 定量）

---

## 约束

- 不讨论具体功能（留到 PRD）
- 不讨论技术实现（留到 Architecture 和 TDD）
- 聚焦「为什么」而非「是什么」或「怎么做」
- 每个断言应尽量有代码中或用户提供的真实信息支撑，若无法确认则标注 `[待确认]`
