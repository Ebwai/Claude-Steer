## 📋 开发要求

### 分支命名规范

| 类型 | 格式 | 示例 |
|------|------|------|
| 功能开发 | `feature/<描述>` | `feature/add-batch-rename` |
| Bug 修复 | `fix/<描述>` | `fix/special-char-crash` |
| 版本发布 | `release/v<版本>` | `release/v1.1.0` |
| 平台开发 | `windows/ubuntu/macos` | `windows` |

### Commit Message 规范

```
<type>: <description>

[optional body]

[optional footer]
```

**Type 类型**：
- `feat`: 新功能
- `fix`: Bug 修复
- `docs`: 文档更新
- `chore`: 其他修改（配置、依赖等）
- `refactor`: 重构
- `test`: 测试相关

**示例**：
```bash
git commit -m "feat: add batch rename feature"
git commit -m "fix: handle special characters in rename"
git commit -m "docs: update README"
git commit -m "chore: update dependencies"
```

### .gitignore 策略

| 分支 | 策略 | 包含内容 |
|------|------|---------|
| develop | 宽松 | knowledge、.claude、文档、宣传材料 |
| release | 严格 | 只有核心代码、README、LICENSE |

## 🔄 日常工作流程

### 1. 功能开发

```bash
# 1. 同步 develop 分支
git checkout develop
git pull origin develop

# 2. 创建功能分支
git checkout -b feature/add-new-feature

# 3. 开发和提交（多次）
git add .
git commit -m "feat: add new feature"

# 4. 推送到远程
git push origin feature/add-new-feature

# 5. 在 GitHub 创建 PR
#    - base: develop ← compare: feature/add-new-feature
#    - 填写描述、关联 issue
#    - 指定 reviewer

# 6. 合并后删除分支
git checkout develop
git pull origin develop
git branch -d feature/add-new-feature
```

### 2. Bug 修复

```bash
# 1. 从 develop 创建修复分支
git checkout develop
git pull origin develop
git checkout -b fix/fix-bug

# 2. 修复和测试
git add .
git commit -m "fix: fix the bug"

# 3. 推送和 PR
git push origin fix/fix-bug

# 4. 合并后删除分支
git checkout develop
git pull origin develop
git branch -d fix/fix-bug
```

### 3. 版本发布

```bash
# 1. 从 develop 创建 release 分支
git checkout develop
git pull origin develop
git checkout -b release/v1.1.0

# 2. 推送到测试仓库
git push origin release/v1.1.0

# 3. 推送到正式仓库（中转）
git push release release/v1.1.0:main

# 4. 在正式仓库打 tag
git tag -a v1.1.0 -m "Release v1.1.0"
git push release v1.1.0

# 5. 在 GitHub 创建 Release
#    - 上传构建产物
#    - 填写更新日志

# 6. 清理 release 分支
git checkout develop
git branch -d release/v1.1.0
git push origin --delete release/v1.1.0
```

### 4. 平台开发

```bash
# 1. 同步平台分支
git checkout windows  # 或 ubuntu、macos
git pull origin windows

# 2. 开发平台特定功能
git add .
git commit -m "feat(windows): add Windows-specific feature"

# 3. 推送到远程
git push origin windows

# 4. 如果需要跨平台，合并到 develop
git checkout develop
git merge windows
git push origin develop
```

## 🚀 新设备设置

### 1. 克隆仓库

```bash
# 克隆测试仓库（日常开发用）
git clone https://github.com/Ebwai/Claude_Steer_Test.git
cd Claude_Steer_Test

# 添加正式仓库作为 remote
git remote add release https://github.com/Ebwai/Claude-Heer.git
```

### 2. 设置分支跟踪

```bash
# 拉取所有分支
git fetch origin
git fetch release

# 切换到 develop 分支（主要开发分支）
git checkout develop
git branch --set-upstream-to=origin/develop develop

# 设置其他分支的跟踪
git checkout windows
git branch --set-upstream-to=origin/windows windows

git checkout ubuntu
git branch --set-upstream-to=origin/ubuntu ubuntu

git checkout macos
git branch --set-upstream-to=origin/macos macos
```

### 3. 验证配置

```bash
# 查看所有分支和跟踪关系
git branch -vv

# 预期输出：
# * develop    xxxxxxx [origin/develop] ...
#   macos      xxxxxxx [origin/macos] ...
#   main       xxxxxxx [origin/main] ...
#   release    xxxxxxx [release/main: ahead N] ...
#   ubuntu     xxxxxxx [origin/ubuntu] ...
#   windows    xxxxxxx [origin/windows] ...
```

## ⚠️ 注意事项

### 1. 提交前检查

```bash
# 查看当前状态
git status

# 查看修改内容
git diff

# 查看提交历史
git log --oneline -5
```

### 2. 避免冲突

```bash
# 开始工作前先同步
git checkout develop
git pull origin develop

# 如果有冲突，解决后继续
git add .
git commit -m "merge: resolve conflicts"
```

### 3. 安全操作

```bash
# 不要直接 push 到 main 或 develop
# 使用 PR 合并

# 不要 force push 到公共分支
# 只在自己的功能分支上使用

# 删除分支前先确认已合并
git branch --merged develop
```
