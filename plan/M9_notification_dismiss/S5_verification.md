## T1 — 实测验证

**测试场景**：

### 场景 1：消息通知界面关闭权限请求

**前置条件**：
- 有活跃的 Claude Code session
- 触发一个权限请求（如文件写入权限）

**测试步骤**：
1. 打开消息通知界面
2. 确认权限请求出现在列表中
3. 点击该请求条目上的"关闭"按钮（✕）
4. 观察：
   - 该请求从列表消失
   - 角标计数 -1
   - Agent 继续等待（不会收到审批响应）

**预期结果**：
- 请求从列表消失
- 角标计数正确更新
- Agent 继续等待，不会自动同意或拒绝

### 场景 2：项目监控界面关闭权限请求

**前置条件**：
- 有活跃的 Claude Code session
- 触发一个权限请求（如文件写入权限）

**测试步骤**：
1. 打开项目监控界面
2. 确认权限请求审批框出现
3. 点击该审批卡片上的"关闭"按钮（✕）
4. 观察：
   - 该审批卡片从列表消失
   - 角标计数 -1
   - Agent 继续等待（不会收到审批响应）

**预期结果**：
- 审批卡片从列表消失
- 角标计数正确更新
- Agent 继续等待，不会自动同意或拒绝

### 场景 3：多次关闭操作

**前置条件**：
- 有活跃的 Claude Code session
- 触发多个权限请求

**测试步骤**：
1. 依次关闭每个权限请求
2. 观察：
   - 每次关闭后，该请求从列表消失
   - 角标计数依次 -1
   - 最终角标计数为 0

**预期结果**：
- 所有请求都能正常关闭
- 角标计数正确更新
- 无残留请求

## T3 — 更新 knowledge/Important_Info.md（如有需要）

**检查内容**：
- 是否需要记录"关闭"操作的特殊行为
- 是否需要记录与现有审批操作的区别

**更新内容**（如果需要）：
```markdown
## [通知关闭机制]

- **关闭操作**：用户点击"关闭"按钮后，通知从列表消失，但 Agent 继续等待（不会收到审批响应）。
- **角标更新**：关闭操作会调用 `NotificationService.decrementBadge()`，角标计数 -1。
- **与审批的区别**：审批操作会发送 TUI 按键序列到 PTY stdin，关闭操作只更新角标，不发送任何按键。
- **IPC 通道**：关闭操作使用 `IPC.PERMISSION_DISMISS`，审批操作使用 `IPC.PERMISSION_RESPOND`。
```
