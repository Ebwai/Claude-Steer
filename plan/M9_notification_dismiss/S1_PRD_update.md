## T1 — 更新概念三：消息通知界面

**位置**：`docs/PRD.md` 第 582-592 行

**当前内容**：
```markdown
#### 概念三：消息通知界面

> 目前只作为多个项目行为批准的统一查看和管理面板（未来会加入更多设计，依然是因为时间问题没有做到最初的设想）。

- **功能描述**：左右分栏的权限请求统一管理面板。左侧为 Agent 请求列表，右侧为消息详情窗口。
- **业务规则与约束**：
  - 请求列表按"项目 → Agent → 任务"层级展示，FIFO 顺序。
  - 权限请求无超时（Agent 一直等待）。
  - 右侧详情：同意 / 不同意 + 输入框 + 发送。
  - 系统通知：桌面通知 + 任务栏角标（跨平台）。
- **数据范围**：权限请求事件（PermissionRequest Hook）。
```

**需要修改的内容**：
在"业务规则与约束"中添加：
```markdown
  - 关闭操作：每个权限请求条目支持点击"关闭"按钮，关闭后通知从列表消失，但 Agent 继续等待（不会收到审批响应）。角标计数同步更新。
```

**修改原因**：
- 原 PRD 只考虑了"审批"操作（同意/拒绝），现在需要支持"关闭"操作
- 关闭操作是纯 UI 操作，不涉及 PTY stdin，Agent 继续等待

## T2 — 更新机制五：系统通知推送

**位置**：`docs/PRD.md` 第 645-652 行

**当前内容**：
```markdown
#### 机制五：系统通知推送
- **功能描述**：将关键事件（当前为权限请求）以桌面通知 + 任务栏角标形式推送，避免用户错过 Agent 阻塞。对应"信息的输出"与"通知统一"。
- **业务规则与约束**：
  - **桌面通知 + 跨平台任务栏角标**（Windows setOverlayIcon / macOS dock.setBadge / Linux setBadgeCount）。
  - **当前硬编码绑定权限请求**：角标计数语义为"待处理权限请求数"（PermissionRequest 触发 notify+increment，审批后 decrement），非通用通知。
  - **设置开关 `desktopNotificationsEnabled` 为死开关**：notify 调用不读此开关，将在 Architecture/TDD 阶段解决（绑定全局设置的桌面通知开关）。
  - **应用内通知队列**（NotificationsPage）与桌面通知是两套，仅通过 permissionRequestsAtom 间接关联。
- **相关代码**：`main/lib/notification/NotificationService.ts`、`main/index.ts:2999-3000`（PermissionRequest 触发）、`:2004`（审批后 decrement）、`renderer/src/atoms/notification.atom.ts`。
- **数据范围**：PermissionRequest Hook 事件。
```

**需要修改的内容**：
在"业务规则与约束"中添加：
```markdown
  - **关闭操作也会更新角标**：用户点击"关闭"按钮后，角标计数 -1（与审批操作相同），但不发送任何按键到 PTY。
```

**修改原因**：
- 关闭操作后，该请求不再"待处理"，角标应该更新
- 需要明确说明关闭操作的角标更新行为
