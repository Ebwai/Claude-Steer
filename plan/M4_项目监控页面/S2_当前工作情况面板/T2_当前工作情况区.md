# T2 — 当前工作情况区（层②）

## 技术原理

层②为整个左半面板的主体区域，`flex: 1; overflow-y: auto`，为每个活跃 Agent/Session 各渲染一个 **Agent Block**。

### 层②整体

**层②是"无限画板"**（CSS `overflow-y: auto`，非 @xyflow，只需 `scrollIntoView` 跟随）
- `flex: 1; overflow-y: auto`，每个 Agent Block 之间虚线分隔（`border: 1px dashed var(--bd)`）

### Agent Block 结构

```
┌─────────────────────────────────────────────┐
│ Block 头部行                                  │
│   🟢 Agent1 [主线程]  Writing · Edit ⟳        │
├────────────────────┬────────────────────────┤
│ 工具大框            │ 经验大框                 │
│ Tools / MCP / CLI  │ Skills / 工作流          │
│ (正在执行=琥珀闪烁) │                          │
├────────────────────┴────────────────────────┤
│ [Subagent 状态块 - 仅有 subagent 时显示]      │
│  紫色边框小卡片：Subagent badge + 名称 + 状态 │
│  内：传入摘要 + 「等待返回...」              │
├──────────────────────────────────────────────┤
│ [活跃 Insight 金色块 - 仅有 insight 时显示]  │
│  ◆ 图标 + insight 文本（金色背景卡片）       │
├──────────────────────────────────────────────┤
│ [Agent 专属消息输入行]                        │
│  Agent标识色 · Agent1  [______________] [↑]  │
├──────────────────────────────────────────────┤
│ ⬛ 打断对话（红色边框按钮）                   │
└──────────────────────────────────────────────┘
```

### 消息输入行规格（PRD 3.2.2 专有）

- 每个 Agent Block 内**各自独立**一个输入框（多 Agent 并行时互不影响）
- 左侧：Agent 标识色色块 + Agent 名称（Agent1=紫色 / Agent2=蓝色 / S-A=绿色 / S-B=天蓝色）
- 输入框：单行（Enter 发送）
- 右侧：`↑` 发送按钮
- 功能：**消息队列机制**——Claude 完成当前响应后（监听 `Stop` hook）自动弹出队列最老的消息注入，不打断心流

### 数据来源

- 工具/经验框实时快照：来自 `PreToolUse`/`PostToolUse` Hook（正在执行=琥珀闪烁 + ⟳）
- 工具框 T/M/C 三列和经验框 S/W 两列：与历史进程线中的节点双框共享相同布局规范
- Subagent 状态块：`SubagentStart` Hook 触发显示，`SubagentStop` Hook 触发消失

### Claude Code 工作状态词

从 statusLine 或 Hook 事件中提取当前工作状态，显示在 Block 头部右侧：
- `Writing` → 正在写入文件
- `Reading` → 正在读取文件
- `Running` → 正在执行命令
- `Thinking` → 正在思考
- `Subagent: pending` → 等待 subagent 返回

## 实现步骤

### 原子目标 1 — activeAgentBlockAtom [x]
文件：`src/renderer/src/atoms/sessions.atom.ts`（追加）
```typescript
interface AgentBlockState {
  sessionId: string
  agentName: string           // 如 "Agent1"
  colorScheme: 'green' | 'blue' | 'purple'
  status: 'Running' | 'Paused' | 'Interrupted'
  workStatusText: string      // 实时工作状态词
  activeTools: ToolEntry[]    // 正在执行的工具列表
  activeExperiences: ExperienceEntry[]
  activeSubagent: SubagentInfo | null
  activeInsight: string | null
}
export const activeAgentBlocksAtom = atom<AgentBlockState[]>([])
```

### 原子目标 2 — 工具/经验实时快照更新 [x]
文件：`src/renderer/src/hooks/useIpcBridge.ts`（扩展）
- 订阅 `PreToolUse` Hook：将工具加入对应 session 的 `activeTools`，`status: 'running'`
- 订阅 `PostToolUse` Hook：更新对应工具状态为 `done`，3s 后从列表移除
- 订阅 `PostToolUseFailure` Hook：更新为 `failed`（红色），5s 后移除

## 验收标准

- 有运行中 session 时，层②显示对应 AgentBlock
- 工具执行时，AgentBlock 工具框中对应条目琥珀色闪烁 + ⟳
- 有 Subagent 运行时，显示紫色边框 Subagent 状态卡片
- 有 Insight 时，显示金色背景 Insight 卡片
- 多 Agent 并行时，每个 Agent 有独立 Block 和独立消息输入框
