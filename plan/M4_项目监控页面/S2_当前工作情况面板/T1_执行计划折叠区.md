# T1 — 执行计划折叠区

## 技术原理

Plan 树从项目目录的 `plan/overall_plan.md` 解析生成树形结构。
使用 `IPC.PLAN_READ` (新增) invoke 读取 plan 文件，解析 `[ ]`/`[x]`/`[!]` 标记。
折叠区高度：折叠态 30px（只显示 header），展开态 200px（左列树 + 右列阅读窗格）。

## 实现步骤

### 原子目标 1 — IPC.PLAN_READ 通道 [ ]
- `ipc-channels.ts`: `PLAN_READ: 'plan:read'`
- `main/index.ts`: 读取 `{projectPath}/plan/overall_plan.md`，返回文件内容字符串

### 原子目标 2 — PlanTree 解析器 [ ]
文件：`src/renderer/src/features/project-monitor/PlanTreeParser.ts`
- 解析 overall_plan.md markdown，提取 `## M`/`### S`/`- [ ] T` 层级
- 输出 `PlanTreeNode[]`：`{ id, level, title, status, children }`

### 原子目标 3 — PlanSection 组件 [ ]
文件：`src/renderer/src/features/project-monitor/PlanSection.tsx`
- 折叠/展开切换（`collapsed` state）
- **折叠态（height: 30px）**：
  - 展开箭头（`▸`）
  - 「执行计划」大写标签
  - 右侧当前 plan 节点 badge（如「M1-S2 · Doing」，紫色背景）
  - 点击标题行触发展开（无需额外按钮）
- **展开态（max-height: 200px）**，内部左右两列（`border-top` 分隔）：

**左列（固定 160px，plan 树）**：
  - 按层级缩进显示全部 M/S/T 节点
  - 每行：展开箭头 + 节点名称 + 状态色块（Done=绿/Doing=琥珀/Todo=灰）
  - 点击某节点 → 高亮选中（蓝紫色背景），**右列同步显示该节点详情**
  - M 级节点名称：紫色加粗；S 级缩进 12px；T 级缩进 22px
  - 状态标记：`[x]`=绿 / `[!]`=橙 / `[ ]`=灰

**右列（阅读窗格，剩余宽度）**：
  - 标题：选中节点编号 + 名称（badge 样式）
  - 内容：当前节点的**完整描述** + 子任务列表
  - 子任务每项前有状态圆点（Done=绿 / Doing=琥珀 / Todo=灰边框）

### 原子目标 4 — selectedPlanNodeAtom [ ]
文件：`src/renderer/src/atoms/plan.atom.ts`（追加）
```typescript
export const selectedPlanNodeAtom = atom<string | null>(null)  // 节点 ID，如 'M1-S2'
```
- 供历史进度拉动条（T8）联动更新

## 验收标准
- 折叠态：30px 高度，显示当前 DOING 节点 badge（如「M1-S2 · Doing」）
- 点击标题展开：左列 Plan 树显示 M/S/T 三级，颜色状态正确
- 点击某个节点：右列显示该节点完整描述和子任务列表
- 拉动条移动时：左列对应节点高亮
