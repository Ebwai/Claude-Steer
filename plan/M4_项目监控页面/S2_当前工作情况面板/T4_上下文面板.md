# T4 — 上下文面板（层④）

## 技术原理

层④在四层布局最底部，`flex-shrink: 0; max-height: 100px; overflow-y: auto`。
实时追踪当前 session 上下文中已加载了哪些组件，并用 tiktoken 估算各组件的 token 占比。

### 上下文组件类型

| 类型标识 | 来源 | 加载时机 |
|---------|------|---------|
| `System` | Claude Code 系统提示 | 始终存在 |
| `CLAUDE.md` | 项目/用户 CLAUDE.md 文件 | 会话开始 |
| `Memory` | auto memory 文件 | 会话开始 |
| `Skills` | 已激活的 Skills | 会话开始 + 使用时 |
| `MCP` | 已配置的 MCP servers | 会话开始 |
| `Rules` | 匹配文件路径的 rules | 读取对应文件时懒加载 |
| `读取文件` | Read/Glob 工具调用的文件 | 每次 Read 调用 |
| `WebFetch` | WebFetch 工具调用 | 每次 WebFetch 调用 |
| `User Input` | 用户输入 | 每次用户提交 |

### Token 估算方式

- 使用 `tiktoken`（npm 包 `tiktoken`）本地估算
- 各组件估算：读取组件内容文件后用 `cl100k_base` 编码计算
- 标注 `≈` 前缀表示估算值
- 记忆压缩（`PostCompact` Hook）后：清除动态组件（读取文件/WebFetch），保留持久组件

## 实现步骤

### 原子目标 1 — contextPanelAtom [x]
文件：`src/renderer/src/atoms/sessions.atom.ts`（追加）
```typescript
interface ContextComponent {
  type: 'System' | 'CLAUDE.md' | 'Memory' | 'Skills' | 'MCP' | 'Rules'
        | 'file' | 'WebFetch' | 'user-input'
  label: string        // 显示名称（文件名/URL title/技能名等）
  url?: string         // 仅 WebFetch 有，作为超链接
  tokenEstimate: number
}
export const contextPanelAtom = atomFamily((sessionId: string) =>
  atom<ContextComponent[]>([])
)
export const selectedContextAgentAtom = atom<string | null>(null)
```

### 原子目标 2 — 上下文追踪服务（Context Tracker）[x]
文件：`src/main/services/ContextTrackerService.ts`
- 订阅 Hook 事件：
  - `SessionStart`：初始化 System/CLAUDE.md/Memory/Skills/MCP 持久组件
  - `PostToolUse`（工具名 Read/WebFetch/Glob/Grep）：追加对应文件/URL 组件
  - `UserPromptSubmit`：追加 User Input 组件
  - `PostCompact`：清除动态组件，保留持久组件
- 通过 IPC `CONTEXT_UPDATED` 推送更新后的组件列表

### 原子目标 3 — Token 估算（可选，后续迭代）[ ]
- 安装 `tiktoken` 包
- `TokenEstimator.ts`：读取组件内容，调用 `cl100k_base` 编码计算 token 数
- 将估算值附加到 `ContextComponent.tokenEstimate`

### 原子目标 4 — ContextPanel 组件 [x]
文件：`src/renderer/src/features/project-monitor/ContextPanel.tsx`
- 层④容器：`flex-shrink: 0; max-height: 100px; overflow-y: auto; border-top: 1px solid var(--bd); padding: 6px`
- 标题栏：「上下文面板」+ token 估算总量（`≈ Ntk`）+ 多 Agent 时显示 Agent 切换按钮
- 内容：`overflow-y: auto`，每行一个组件条目
- 每行格式：`- 组件类型 · 文件名/描述`（font-size: 10px）
- WebFetch 条目：文件名作为超链接（`<a href={url} target="_blank">`）

### 原子目标 5 — 多 Agent 切换 [x]
- 多 Agent 时显示切换按钮：`[Agent1] [Agent2]`，当前选中高亮
- 切换后更新 `selectedContextAgentAtom`，展示对应 agent 的上下文列表

## 新增 IPC 通道

- `CONTEXT_UPDATED` (push): `{ sessionId, components: ContextComponent[] }` — 上下文更新通知

## 验收标准

- 启动 session 后层④出现持久组件（System/CLAUDE.md）
- 执行 Read 工具后，读取的文件名立即出现在组件列表
- 执行 WebFetch 后，URL 以超链接形式出现
- 记忆压缩后（/compact），动态组件清除，持久组件保留
- 多 Agent 时可切换 Agent 查看各自上下文
