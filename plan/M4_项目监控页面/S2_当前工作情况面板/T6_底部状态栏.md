# T6 — 底部状态栏

## 技术原理

底部状态栏位于四层布局（层①-④）之外的最底部，`height: ~20px`，全局固定在当前工作情况面板底端。

```
> Writing: Beaming  —  「正在分析项目结构，准备编写接口层」 ⟳
```

数据来源：
- 工作状态词：从 statusLine `model` 字段 + 最新 Hook 事件推断当前操作类型
- 实时任务摘要：最新一条 assistant 文本输出的前 60 字（搞怪/活泼风格的第一句）

## 实现步骤

### 原子目标 1 — statusBarAtom [ ]
文件：`src/renderer/src/atoms/sessions.atom.ts`（追加）
```typescript
interface StatusBarState {
  workStatus: string      // 如 "Writing" / "Reading" / "Subagent: pending"
  taskSummary: string     // 实时任务摘要（最新 assistant 文本前 60 字）
  agentCount: number      // 并行 Agent 数量
}
export const statusBarAtom = atom<StatusBarState>({
  workStatus: '',
  taskSummary: '',
  agentCount: 0
})
```

### 原子目标 2 — StatusBar 组件 [ ]
文件：`src/renderer/src/features/project-monitor/StatusBar.tsx`
- `height: 20px; flex-shrink: 0; border-top: 1px solid var(--bd); padding: 0 8px`
- `display: flex; align-items: center; gap: 8px; font-size: 10px; color: var(--tx2)`
- 左侧：`>` 符号（高亮色）+ 工作状态词（颜色根据类型变化）
- 中间：`—`分隔 + 任务摘要（截断，`white-space: nowrap; overflow: hidden; text-overflow: ellipsis`）
- 右侧：`⟳` 旋转图标（仅 Running 状态时显示）
- 多 Agent 并行时：替换为「N Agents 并行」+ 最活跃 Agent 的状态

### 原子目标 3 — 工作状态词映射 [ ]
文件：`src/renderer/src/features/project-monitor/StatusBar.tsx`（或独立 utils）
```typescript
const WORK_STATUS_MAP: Record<string, string> = {

```

### 原子目标 4 — 实时任务摘要更新 [ ]
- 监听 `timelineBySessionAtom` 中最新的 assistant 节点文本
- 取前 60 字作为摘要（裁掉 markdown 格式标记）

## 验收标准

- Session 运行时底部出现状态词（如 Writing）
- 状态词随工具调用实时变化
- 右侧任务摘要显示最新 assistant 输出的开头内容
- 多 Agent 并行时显示「N Agents 并行」
