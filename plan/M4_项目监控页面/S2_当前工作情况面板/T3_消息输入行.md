# T3 — 消息输入行

## 技术原理

消息输入行允许用户向当前运行的 Claude session 发送消息。
底层通过 `IPC.SESSION_INPUT` 写入 PTY stdin（已实现）。
队列机制：多条消息入队，Stop Hook 触发时 FIFO 自动注入（通过 `MessageQueue` 类，已在 M2 实现）。
打断按钮：`IPC.SESSION_STOP` 终止当前运行的 PTY 进程。

### 原子目标 3 — 打断按钮完整流程 [ ]

> 目标：结束 Claude Code 会话，断开 PTY，解绑 IPC 映射

**按下后完整流程**（渲染层发起，主进程执行）：

1. 渲染层：调用 `IPC.SESSION_STOP { sessionId: ptySessionId }`
2. 主进程 SESSION_STOP handler：
   a. 通过 `PtyManager` 找到对应 PTY 实例
   b. 连续向 PTY stdin 写入两次 `\x03`（Ctrl+C）—— 终止正在运行的 Claude Code 进程
   c. 等待 500ms（给 Claude Code 时间响应 Ctrl+C 并退出）
   d. 调用 `PtyManager.stopSession(ptyId)` —— 关闭 PTY 进程本身（`pty.kill()`）
   e. **不手动调用 `unbindPtyFromClaudeSession`**：PTY 退出后 Claude Code 会自然发 SessionEnd hook，由 hook 事件触发解绑和状态变更
3. 渲染层 `useIpcBridge` SessionEnd handler：
   - `activeSessionsAtom`：将对应 session 状态更新为 `Interrupted`（不从 Map 移除，历史进程线保留）
   - LeftPanel AgentBlock 消失（`projectSessions` 过滤器只保留 Running/Paused）
4. 主进程 SessionEnd hook：`unbindPtyFromClaudeSession(claudeId)` → 推送 `PTY_UNBIND`
5. 渲染层 `PTY_UNBIND` handler：**仅清 `ptyBindingsAtom` 绑定表，不改 session 状态**

**关键约束**：
- 两次 Ctrl+C 之间无需间隔（连续写入）
- 500ms 等待后才调用 `pty.kill()`，避免强杀前 JSONL 未写完
- **`PTY_UNBIND` handler 不能修改 session 状态**：branch 触发时父 PTY 也会经历 PTY_UNBIND（旧 claudeId 解绑），若此时改 session 状态，branch 后父 PTY 对应的 AgentBlock 会立刻消失

文件：`src/main/index.ts` SESSION_STOP handler（修改：加 2×Ctrl+C + 500ms delay）+ `useIpcBridge.ts` PTY_UNBIND handler（维持现状，不加 status 变更）
