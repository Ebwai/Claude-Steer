# T5 — 请求交互框（层③）

## 技术原理

层③在四层布局中间（层②和层④之间），`flex-shrink: 0`，仅当有权限请求时显示。

当 Claude Code 发出 `PermissionRequest` Hook 时，在此区域以**阻断式**方式展示审批控件。
用户点击同意/拒绝后，通过 PTY stdin 管道写入对应的响应。

### PermissionRequest Hook 数据

```json
{
  "hook_event_name": "PermissionRequest",
  "session_id": "...",
  "tool_name": "Bash",
  "tool_input": { "command": "rm -rf ./temp" },
  "description": "是否允许执行：rm -rf ./temp"
}
```

### 响应机制

- **同意**：向 PTY stdin 写入 `y\n`（或 `yes\n`），可选附加文字
- **拒绝**：向 PTY stdin 写入 `n\n`，可选附加原因
- 附加文字：可选输入框，输入后一并发送
- 多个 Agent 同时有请求：FIFO 堆叠显示多条

## 实现步骤

### 原子目标 1 — permissionRequestsAtom [ ]
文件：`src/renderer/src/atoms/sessions.atom.ts`（追加）
```typescript
interface PermissionRequest {
  requestId: string
  sessionId: string
  agentName: string
  toolName: string
  description: string
  receivedAt: number
}
export const permissionRequestsAtom = atom<PermissionRequest[]>([])
```

### 原子目标 2 — PermissionRequest Hook 处理 [ ]
文件：`src/main/services/SessionService.ts`（扩展）或 `hook-server` handler
- 接收 `PermissionRequest` Hook → 通过 IPC `NOTIFICATION:new` 推送到渲染进程
- 同时触发桌面通知（N1）

### 原子目标 3 — RequestApprovalPanel 组件 [ ]
文件：`src/renderer/src/features/project-monitor/RequestApprovalPanel.tsx`
- 仅 `permissionRequestsAtom` 非空时渲染（`flex-shrink: 0`）
- 每条请求显示：
  - 来源：「[Agent名称] 请求执行：{操作描述}」（`font-size: 11px`）
  - **同意** 按钮（绿色）+ **拒绝** 按钮（红色）
  - 可选输入框（`placeholder: "附加信息（可选）"`，最初折叠，点击按钮后弹出）
  - 点击同意/拒绝 → 展开输入框 → 点击「发送」，向 PTY stdin 写入响应
- 多条请求按 FIFO 顺序垂直堆叠

### 原子目标 4 — PTY 响应注入 [ ]
文件：`src/main/services/SessionService.ts`
- 新增 `respondToPermission(sessionId, approved: boolean, message?: string)` 方法
- 写入 PTY stdin：`${approved ? 'y' : 'n'}${message ? '\n' + message : ''}\n`
- 注入后从 `permissionRequestsAtom` 中移除该条请求

### 原子目标 5 — 消息通知页面同步 [ ]
- 同一个 `permissionRequestsAtom` 供消息通知页面（M5-S1）读取
- 在消息通知页面处理请求后，同样更新此 atom（两处 UI 共享同一状态）

## 新增 IPC 通道

- `PERMISSION_RESPOND` (invoke): `{ sessionId, requestId, approved: boolean, message?: string }` → `{ ok }`

## 验收标准

- Claude 请求执行危险命令时，层③出现审批控件
- 点击同意 + 可选附加信息 + 发送：Claude 收到 `y\n{message}` 后继续执行
- 点击拒绝：Claude 收到 `n\n` 后取消操作
- 多个 Agent 同时请求时，多条审批控件堆叠显示
- 处理后对应条目消失
