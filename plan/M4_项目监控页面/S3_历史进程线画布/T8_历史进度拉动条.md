# T8 — 历史进度拉动条

> **更新说明（2026-04-21 v2）**：基于架构实测深度重写。修正了以下关键错误：
> - `projects.json` 的 `sessionIds` 存的是 ptyId，与 JSONL 文件名（claudeId）完全不同，无法用于查找历史文件
> - `agentLabelsAtom` 只覆盖 `ptySessionIds` 中的实时 session，历史 session 需要独立标签派生
> - `forkedFrom` 字段（JSONL 第一行）是恢复历史 branch 关系的唯一可靠来源
> - subagent JSONL 存在子目录 `<parentId>/subagents/<agentId>.jsonl`，需一并加载

---

## 技术背景

### JSONL 文件发现策略（与 projects.json 无关）

Claude Code 的 `~/.claude/projects/<encoded-path>/` 目录下：
- 顶层 `.jsonl` 文件名 = **claudeId**（永久 UUID），内部第一条有效行含 `cwd` 和可选的 `forkedFrom` 字段
- `projects.json` 的 `sessionIds` 存 **ptyId**（会变），与 JSONL 文件名毫无关系，**禁止用于文件发现**
- **正确方式**：扫描目录所有 `.jsonl`，读第一行的 `cwd` 字段过滤归属

### branch 关系恢复

JSONL 第一行的 `forkedFrom` 字段（若存在）：
```json
{ "forkedFrom": { "sessionId": "<父claudeId>", "messageUuid": "<分叉点消息UUID>" } }
```
这是恢复历史 `sessionRelationsAtom` 的唯一来源。`messageUuid` 可用于计算 `triggerNodeIndex`（branch 连线起点的 Y 偏移）。

### subagent JSONL 结构

```
~/.claude/projects/<encoded>/
  <parentClaudeId>/
    subagents/
      agent-<agentId>.jsonl   ← agentId = 文件名去掉 "agent-" 前缀
```
subagent JSONL 内部 `sessionId` = 父 session 的 claudeId，`agentId` = 子 agent 标识。

### 数据流

```
用户切换到某项目标签（activeProjectIdAtom 变化）
  ↓
历史加载 Hook（useHistoryLoader）触发
  ↓
主进程 IPC.PROJECT_HISTORY_SCAN：扫描 ~/.claude/projects/<encoded>/，
  读每个 .jsonl 第一行 cwd 过滤，按 mtime 降序排列，取最近 MAX_HISTORY_SESSIONS 个
  返回：SessionHistoryMeta[]（claudeId, cwd, transcriptPath, mtime, forkedFrom?）
  ↓
renderer 层按 mtime 升序（时间正序）构建 Session 对象写入 activeSessionsAtom
  （仅写入 claudeId 尚不存在的 entry，防覆盖实时 session）
  ↓
恢复 sessionRelationsAtom：有 forkedFrom 的 → 写 branch 关系，
  计算 triggerNodeIndex（父 JSONL 中 forkedFrom.messageUuid 的行索引）
  ↓
ProcessLineCanvas 渲染 SessionFrameNode（layout 自动处理 cluster）
  ↓
ProcessTimeline 挂载后触发 IPC.JSONL_WATCH（readFromStart=true）加载历史记录
  （subagent 通过父 session 子目录同步加载）
  ↓
画布视口 → fitView 到最新 session 框（时间最新的 mtime 对应的 node.id）
```

### 原子目标 1 — 主进程 PROJECT_HISTORY_SCAN handler [x]

**文件**：`src/main/index.ts`（在 IPC handlers 区追加）

```typescript
ipcMain.handle(IPC.PROJECT_HISTORY_SCAN, async (
  _event,
  payload: { projectPath: string; maxSessions: number }
) => {
  const encoded = payload.projectPath.replace(/[/\\:]/g, '-')
  const claudeDir = path.join(os.homedir(), '.claude', 'projects', encoded)
  const MAX = payload.maxSessions ?? 20

  if (!fs.existsSync(claudeDir)) return []

  const results: SessionHistoryMeta[] = []

  try {
    const entries = fs.readdirSync(claudeDir)

    for (const entry of entries) {
      // 顶层 .jsonl 文件（claudeId = 文件名去扩展名）
      if (!entry.endsWith('.jsonl')) continue

      const filePath = path.join(claudeDir, entry)
      const claudeId = entry.replace('.jsonl', '')

      try {
        const stat = fs.statSync(filePath)
        const content = fs.readFileSync(filePath, 'utf-8')
        const lines = content.split('\n').filter((l) => l.trim())

        // 找第一行含 cwd 的记录
        let cwd: string | undefined
        let forkedFrom: { sessionId: string; messageUuid: string } | null = null

        for (const line of lines) {
          try {
            const obj = JSON.parse(line) as Record<string, unknown>
            if (obj['cwd']) {
              cwd = obj['cwd'] as string
              const ff = obj['forkedFrom'] as { sessionId: string; messageUuid: string } | undefined
              forkedFrom = ff ?? null
              break
            }
          } catch { continue }
        }

        if (!cwd) continue  // 无 cwd 的 JSONL 跳过（可能是损坏文件）

        // 只保留匹配项目路径的 session
        if (cwd !== payload.projectPath && !cwd.startsWith(payload.projectPath + '/')) continue

        // 扫描 subagent 子目录
        const subagentFiles: Array<{ agentId: string; filePath: string }> = []
        const subagentDir = path.join(claudeDir, claudeId, 'subagents')
        if (fs.existsSync(subagentDir)) {
          for (const sf of fs.readdirSync(subagentDir)) {
            if (!sf.startsWith('agent-') || !sf.endsWith('.jsonl')) continue
            const agentId = sf.replace('agent-', '').replace('.jsonl', '')
            subagentFiles.push({ agentId, filePath: path.join(subagentDir, sf) })
          }
        }

        results.push({
          claudeId,
          cwd,
          transcriptPath: filePath,
          mtime: stat.mtimeMs,
          forkedFrom,
          subagentFiles,
        })
      } catch { continue }
    }
  } catch (err) {
    console.error('[ipc] project:history-scan failed:', err)
    return []
  }

  // 按 mtime 降序排列，取最近 MAX 个，再按 mtime 升序（时间正序）返回
  results.sort((a, b) => b.mtime - a.mtime)
  const selected = results.slice(0, MAX)
  selected.sort((a, b) => a.mtime - b.mtime)

  console.log(`[ipc] project:history-scan: found ${results.length} sessions, returning ${selected.length} for ${payload.projectPath}`)
  return selected
})
```

**注意**：只读操作，无需原子写入。

### 原子目标 2 — `agentLabelsAtom` 扩展：覆盖历史 session [x]

**文件**：`src/renderer/src/atoms/agentLabels.atom.ts`

当前实现只覆盖 `ptySessionIds` 中的实时 session，历史 `Completed` session 不在其中。需改为覆盖所有 `activeSessionsAtom` 中的 session：

```typescript
export const agentLabelsAtom = atom<Map<string, string>>((get) => {
  const activeSessions = get(activeSessionsAtom)
  const ptySessionIds  = get(ptySessionIdsAtom)
  const sessionRelations = get(sessionRelationsAtom)

  const allSessions = Array.from(activeSessions.values())

  // 实时 session：Running / Paused，且在 ptySessionIds 中
  const liveSessions = allSessions
    .filter((s) => ptySessionIds.has(s.claudeId ?? s.id) && (s.status === 'Running' || s.status === 'Paused'))
    .sort((a, b) => a.startedAt - b.startedAt)

  // 历史 session：Completed / Interrupted（或未在 ptySessionIds 中的）
  const histSessions = allSessions
    .filter((s) => !ptySessionIds.has(s.claudeId ?? s.id))
    .sort((a, b) => a.startedAt - b.startedAt)

  const labels = new Map<string, string>()

  // 实时 session 标签（原有逻辑不变）
  const liveMain = liveSessions.filter((s) => !sessionRelations.has(s.claudeId ?? s.id))
  const liveBranch = liveSessions.filter((s) => sessionRelations.has(s.claudeId ?? s.id))
  liveMain.forEach((s, idx) => labels.set(s.claudeId ?? s.id, idx === 0 ? '主线' : `Agent${idx + 1}`))
  liveBranch.forEach((s, idx) => labels.set(s.claudeId ?? s.id, idx === 0 ? 'Branch' : `Branch${idx + 1}`))

  // 历史 session 标签（独立编号，不与实时混用）
  const histMain = histSessions.filter((s) => !sessionRelations.has(s.claudeId ?? s.id))
  const histBranch = histSessions.filter((s) => sessionRelations.has(s.claudeId ?? s.id))
  histMain.forEach((s, idx) => labels.set(s.claudeId ?? s.id, idx === 0 ? '历史主线' : `历史 Agent${idx + 1}`))
  histBranch.forEach((s, idx) => labels.set(s.claudeId ?? s.id, idx === 0 ? '历史 Branch' : `历史 Branch${idx + 1}`))

  return labels
})
```

### 原子目标 3 — `useHistoryLoader` Hook [x]

**文件**：`src/renderer/src/hooks/useHistoryLoader.ts`（新建）

监听 `activeProjectIdAtom` 变化，触发历史加载。

```typescript
export const MAX_HISTORY_SESSIONS = 20

export function useHistoryLoader(): void {
  const store = useStore()

  useEffect(() => {
    const unsub = store.sub(activeProjectIdAtom, async () => {
      const projectId = store.get(activeProjectIdAtom)
      if (!projectId) return

      const project = store.get(projectByIdAtom(projectId))
      if (!project) return

      console.log(`[HistoryLoader] Loading history for project: ${project.path}`)

      try {
        const metas = await window.api.invoke(IPC.PROJECT_HISTORY_SCAN, {
          projectPath: project.path,
          maxSessions: MAX_HISTORY_SESSIONS,
        }) as SessionHistoryMeta[]

        if (!metas.length) return

        // ── 写入 activeSessionsAtom（只写不存在的 entry，防覆盖实时 session） ──
        store.set(activeSessionsAtom, (prev) => {
          const next = new Map(prev)
          for (const meta of metas) {
            if (next.has(meta.claudeId)) continue  // 实时 session 优先
            next.set(meta.claudeId, {
              id: meta.claudeId,      // 无 ptyId，用 claudeId 占位
              claudeId: meta.claudeId,
              projectId: project.id,
              status: 'Completed',
              currentModel: null,
              tokenUsage: { current: null, max: null, usedPercentage: null },
              transcriptPath: meta.transcriptPath,
              cwd: meta.cwd,
              startedAt: meta.mtime,  // 用 mtime 近似 startedAt（历史 session 无法精确获取）
              endedAt: meta.mtime,
              worktreePath: null,
            })
          }
          return next
        })

        // ── 恢复 branch 关系 ─────────────────────────────────────────────────
        store.set(sessionRelationsAtom, (prev) => {
          const next = new Map(prev)
          for (const meta of metas) {
            if (!meta.forkedFrom) continue
            if (next.has(meta.claudeId)) continue  // 实时 branch 关系不覆盖
            next.set(meta.claudeId, {
              type: 'branch',
              parentSessionId: meta.forkedFrom.sessionId,
              side: 'right',           // 默认右侧，branchRegistry 运行时会修正
              lineLength: 40,          // 默认最小间距
              triggerNodeIndex: 0,     // 简化：历史 branch 暂用 0，不精确计算消息索引
            })
          }
          return next
        })

        // ── 视口跳转：fitView 到 mtime 最新的 session ────────────────────────
        const latestMeta = metas[metas.length - 1]  // 已按 mtime 升序排列
        store.set(focusedSessionIdAtom, latestMeta.claudeId)
        store.set(viewportModeAtom, 'focus')

        console.log(`[HistoryLoader] Loaded ${metas.length} history sessions, focusing ${latestMeta.claudeId.slice(0, 8)}`)
      } catch (err) {
        console.error('[HistoryLoader] Failed to load history:', err)
      }
    })

    return unsub
  }, [store])
}
```

调用位置：在 `App.tsx` 或 `ProjectMonitorPage.tsx` 中调用 `useHistoryLoader()`。

```typescript
// 同步加载 subagent 子目录（仅历史模式）
if (payload.readFromStart) {
  const subagentDir = path.join(path.dirname(payload.transcriptPath), payload.sessionId, 'subagents')
  // 注意：transcriptPath = ~/.claude/projects/<enc>/<claudeId>.jsonl
  //       子目录 = ~/.claude/projects/<enc>/<claudeId>/subagents/
  const subDir2 = path.join(
    path.dirname(payload.transcriptPath),
    path.basename(payload.transcriptPath, '.jsonl'),
    'subagents'
  )
  if (fs.existsSync(subDir2)) {
    for (const sf of fs.readdirSync(subDir2)) {
      if (!sf.startsWith('agent-') || !sf.endsWith('.jsonl')) continue
      const agentId = sf.replace('agent-', '').replace('.jsonl', '')
      // 读取 subagent JSONL 并分批推送
      try {
        const subContent = fs.readFileSync(path.join(subDir2, sf), 'utf-8')
        const subLines = subContent.split('\n').filter((l) => l.trim())
        const subRecords = subLines.map((l) => parseJsonlLine(l)).filter(Boolean)
        for (let i = 0; i < subRecords.length; i += BATCH_SIZE) {
          // 用 JSONL_SUBAGENT_RECORD 格式推送（复用现有 jsonlHandler 逻辑）
          for (const rec of subRecords.slice(i, i + BATCH_SIZE)) {
            mainWindow?.webContents.send(IPC.JSONL_SUBAGENT_RECORD, {
              sessionId: payload.sessionId,
              agentId,
              record: rec,
            })
          }
        }
        console.log(`[ipc] jsonl:watch subagent ${agentId}: sent ${subRecords.length} records`)
      } catch (e) {
        console.warn(`[ipc] jsonl:watch subagent read failed: ${String(e)}`)
      }
    }
  }
}
```

## 验收标准

- [ ] 切换到已有历史的项目标签 → 历史面板展示最近 20 个 session 框（按时间从左到右或按布局排列）
- [ ] 历史 branch session → 有连线，位于父框旁侧
- [ ] subagent 历史 → 进程线中显示 SubagentBlock 插入线
- [ ] 视口自动跳转到最新（mtime 最大）的 session 框
- [ ] 每个 SessionFrameNode 右侧出现 16px 宽拉动条
- [ ] 拖动滑块 → `ptl-container` 连续跳转到对应 user_input 节点
- [ ] 键盘 ↓/↑ → 精确移动一个 user_input 区间
- [ ] 实时运行 session → 滑块自动锁定底部跟随
- [ ] 切换回有运行中 session 的项目 → follow 模式恢复，历史加载不影响实时 session

## 已知限制与后续优化

- 历史 branch 的 `triggerNodeIndex`（连线 Y 偏移）暂定为 0，需后续通过解析父 JSONL 精确计算
- `startedAt` 用文件 mtime 近似，精度约为最后修改时间（非会话开始时间）
- 超过 20 个历史 session 的项目（如 Test2 有 312 个）只展示最近 20 个，更早历史暂不支持分页加载
