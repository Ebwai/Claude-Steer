# T4 — 十类插入元素 A-J

## 技术原理

### 插入元素模型（LineInsertion）

每个 Claude 回复节点之间，根据 Hook/JSONL 触发情况动态插入十类元素。
统一数据结构（`src/shared/types/lineInsertion.ts`）：

```typescript
interface LineInsertion {
  type: 'tool' | 'mcp' | 'cli' | 'skill' | 'workflow' | 'insight'
        | 'subagent' | 'branch' | 'btw' | 'user-input'
  direction: 'left' | 'right'          // 时间轴插入方向
  color: '#e6430d' | '#DA7756'          // 工具类右侧=橙红；体验/交互类左侧=暖橙
  length: 'short' | 'medium' | 'long'  // 短=1单位/中=2单位/长=3单位
  sessionId: string
  timestamp: number
  badgeContent: Record<string, string>  // 工具名、操作描述等
  status: 'pending' | 'running' | 'done' | 'failed'
  isAnimating: boolean                  // 琥珀色闪烁 + ⟳ 后缀（正在执行时）
  lineLabel?: string                    // 插入线上的文字（H/I/J 类有值，如"继承记忆"/"临时提问"）
}
```

### 十类规格总表

| 类别 | 颜色 | 方向 | 长度 | 识别来源 |
|------|------|------|------|---------|
| A. Tools 调用 | `#e6430d` | 右侧 | 短 | `PreToolUse`/`PostToolUse` Hook，工具名非 `mcp__` 前缀且非 CLI skill |
| B. MCPs 调用 | `#e6430d` | 右侧 | 中 | `PreToolUse` Hook，工具名含 `mcp__` 前缀 |
| C. CLI 调用 | `#e6430d` | 右侧 | 长 | Skill 工具调用，且 skill 名包含 `cli`（大小写不敏感），来自 `ToolCategoryResolver` |
| D. Skills 调用 | `#DA7756` | 左侧 | 短 | `Skill` 工具调用（排除 CLI 分类），JSONL `tool_use.name = 'Skill'` |
| E. 工作流调用 | `#DA7756` | 左侧 | 中 | Hook 事件触发，来自 settings.json hooks 配置（非 claude 内部工具调用） |
| F. Insight 产生 | `#DA7756` | 左侧 | 长 | `InsightExtractor` 从 assistant 文本中提取（正则匹配 `★ Insight` 或 backtick 块） |
| G. Subagent 调用 | `#DA7756` | 左侧 | 长 | `SubagentStart` Hook→分配线（**线上文字"分配任务"**，badge:subagent名称+给subagent的信息）；`SubagentStop` Hook→返回线（**线上文字"返回任务结果"**，badge:返回内容摘要）；含 mini 进程线 |
| H. /branch | `#DA7756` | 左侧 | 长 | `UserPromptSubmit` Hook + 正则匹配 `/branch`；**线上文字"继承记忆"**；badge:"branch：来自于Agentxx的分支探索" |
| I. /btw | `#DA7756` | 左侧 | 长 | `UserPromptSubmit` Hook + 正则匹配 `/btw`；**线上文字"临时提问"**；badge: 用户/btw输入内容前20字 + 展开按钮 |
| J. 用户输入 | `#DA7756` | 左侧 | 长 | `UserPromptSubmit` Hook；**线上文字"继承记忆"**；badge: 用户输入前20字 + 展开按钮 |

### 工具调用分类识别（ToolCategoryResolver）

文件：`src/main/lib/jsonl/ToolCategoryResolver.ts`
- 启动时扫描 `~/.claude/skills/` 和项目级 `.claude/skills/` 目录
- 将名称含 `cli`（大小写不敏感）的 skill 标记为 CLI 类型，放入 `cliSkillNames: Set<string>`
- 分类逻辑：
  1. 工具名含 `mcp__` → B 类 MCP
  2. 工具名为 `Skill` 且 input.skill 在 `cliSkillNames` 中 → C 类 CLI
  3. 工具名为 `Skill` → D 类 Skills
  4. 其他内置工具 → A 类 Tools

### Insight 文本提取（InsightExtractor）

文件：`src/main/lib/jsonl/InsightExtractor.ts`
- 监听 JSONL `role=assistant` 文本，正则匹配 insight 格式（`` `★ Insight ─────` `` 标记块）
- 解析后通过事件总线广播 `InsightFoundEvent`
- ⚠️ 该方案为 MVP 初步方案，需实践验证后记录到 `Important_Info.md`

- `.ins-line.left`：从时间轴向左延伸（`margin-right: 0`，使用负 margin）

- `@keyframes amber-blink`：背景色在 `var(--amber)` 和透明之间闪烁

## 新增 IPC 通道

- `LINE_INSERTION` (push): `{ sessionId, insertion: LineInsertion }` — 主进程推送单条插入元素
- `INSIGHT_FOUND` (push): `{ sessionId, text: string, timestamp: number }` — Insight 提取完成

## 验收标准

- 执行 Bash 工具调用 → 右侧插入橙红色短线 + badge 显示工具名和描述
- 调用 MCP 工具 → 右侧插入橙红色中线
- 调用 Skill → 左侧插入暖橙短线
- 有 Insight 文本 → 左侧插入暖橙长线 + badge 显示前 20 字
- Subagent 启动 → 左侧插入暖橙长线 + mini 进程线折叠块
- 正在执行的条目显示琥珀色闪烁 + ⟳ 后缀
