## 一、UI 交互设计

### 1.1 assistant 节点底部操作栏

每个 `assistant` 节点卡片底部追加一个 `ptl-action-bar`（高度 22px），根据节点状态显示不同按钮：

```
┌─────────────────────────────────────────────────┐
│  Claude 回复内容…                               │
├─────────────────────────────────────────────────┤  ← ptl-action-bar
│  [☁ Git 快照]  [⊕ 创建多 Agent ▸]              │  ← 最后节点（isLastNode=true）
└─────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────┐
│  Claude 回复内容…                               │
├─────────────────────────────────────────────────┤
│  [↩ 回退到此版本]                               │  ← 已 git 非最后节点
└─────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────┐
│  Claude 回复内容…                               │
├─────────────────────────────────────────────────┤
│  （空，无可用操作）                             │  ← 普通节点
└─────────────────────────────────────────────────┘
```

**操作栏按钮显示规则**（互斥分支）：

| 条件 | 显示按钮 |
|------|---------|
| `isLastNode === true` | ☁ Git 快照 + ⊕ 创建多 Agent ▸（含子菜单） |
| `isLastNode === false && isGitted === true` | ↩ 回退到此版本 |
| `isLastNode === false && isGitted === false` | 操作栏存在但为空（占位 22px，保持视觉一致性） |

> **设计理由**：最后节点提供"前进"操作（快照 / 分支）；历史节点只在已 git 时提供"时间旅行"（回退）。这两类操作语义互斥，不会同时出现。

### 1.2 user_input 节点——预留删除位置

user_input 节点气泡右上角预留一个 `ptl-delete-placeholder` 按钮区（`opacity: 0.3; cursor: not-allowed`），hover 显示 tooltip：「删除功能即将上线」。

**预留形态**：灰色 🗑 图标，点击无响应。

**删除功能设计说明（待未来实现时评审）**：
- 删除区间定义：从当前 `user_input` 节点（含）到下一个 `user_input` 节点（不含）之间的全部 JSONL 记录
- 即删除「一次完整的用户输入 + Claude 回复 + 工具调用」的完整对话轮次
- 若区间内有 git commit：需评估是否用 `git rebase --onto <base> <start> <end>`（非交互式，严禁 `git rebase -i`）
- **此功能不在本 T9 中实现，只预留 UI 位置**

### 1.3 ⊕ 创建多 Agent 子菜单（三个选项）

点击「⊕ 创建多 Agent ▸」展开内联子菜单（不用浮层，直接展开在操作栏下方）：

## 二、数据层设计

### 2.1 TimelineNode 新增 git 状态字段

**文件**：`src/renderer/src/atoms/timeline.atom.ts`

```typescript
export interface TimelineNode {
  // ... 现有字段 ...

  /** 该节点是否已被 git 快照（commit 成功后设为 true）*/
  isGitted?: boolean
  /** 该节点对应的 git commit hash（git 快照成功后写入）*/
  commitHash?: string
}
```

> `isGitted` 和 `commitHash` 在 git 快照成功后由 IPC 返回结果写入对应节点。

### 2.2 isLastNode 计算

`isLastNode` **不存入 atom**，由 `ProcessTimeline` 渲染时动态计算：

```typescript
// 当前 assistant 节点是否是 visibleNodes 中最后一个 assistant 节点
const lastAssistantIdx = [...visibleNodes].reverse().findIndex((n) => n.type === 'assistant')
const isLastNode = lastAssistantIdx >= 0
  ? visibleNodes[visibleNodes.length - 1 - lastAssistantIdx].id === node.id
  : false
```

**理由**：`isLastNode` 是渲染时的相对位置，随节点增减动态变化，不适合持久化到 atom。

### 2.3 git 节点颜色

已 git 节点的圆点颜色改为深色标记（区别于未 git 节点）：

```typescript
// 圆点颜色规则（在 TimelineNodeItem 中）
const dotColor = node.isGitted
  ? '#1a7a6e'   // 已 git：深蓝绿（固定色，不随 session 颜色变化）
  : undefined   // 未 git：使用 CSS 默认色
```

### 原子目标 6 — 主进程 GIT_SNAPSHOT handler [x]

**文件**：`src/main/index.ts`

```typescript
ipcMain.handle(IPC.GIT_SNAPSHOT, async (_event, payload: { projectPath: string; message: string }) => {
  try {
    const { execFileSync } = await import('child_process')
    execFileSync('git', ['-C', payload.projectPath, 'add', '-A'])
    const hash = execFileSync(
      'git', ['-C', payload.projectPath, 'commit', '-m', payload.message],
      { encoding: 'utf-8' }
    ).match(/\[[\w\s]+\s([0-9a-f]+)\]/)?.[1] ?? ''
    console.log(`[git:snapshot] committed ${hash} for ${payload.projectPath}`)
    return { ok: true, commitHash: hash }
  } catch (err) {
    console.error('[git:snapshot] failed:', err)
    return { ok: false, error: String(err) }
  }
})
```

ipcMain.handle(IPC.GIT_RESET_HARD, async (_event, payload: { projectPath: string; commitHash: string }) => {

    execFileSync('git', ['-C', payload.projectPath, 'reset', '--hard', payload.commitHash])

})
