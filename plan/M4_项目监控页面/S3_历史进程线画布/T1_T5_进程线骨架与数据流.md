# T1~T5 — 进程线骨架与数据流（合并实现）

## 技术原理

### JSONL → 进程线节点映射
JSONL 文件每行对应 `JsonlRecord`（已由 `JsonlParser` 解析）。
进程线以"用户输入"为分段：每次 `type=user` 消息产生一个新段，该段内包含：
- 用户输入标签（蓝色）
- Claude 回复文本卡片（左侧）
- 工具调用列表（右侧蓝色 Tools 框）

### 数据流（双模式）

**历史模式**（打开已有项目）：
1. 渲染进程调用 `JSONL_WATCH`（新 IPC）：`{ projectPath, sessionId, readFromStart: true }`
2. 主进程批量读取 JSONL → 分批推送 `JSONL_RECORDS`（新 IPC push event）到渲染进程
3. 渲染进程更新 `timelineNodesAtom`

**实时模式**（运行中 session，statusLine 返回 transcript_path 后）：
1. 渲染进程调用 `JSONL_WATCH`：`{ projectPath, sessionId, readFromStart: false }`
2. 主进程启动 `JsonlWatcher.watchFile(transcript_path, sessionId, false)`
3. 主进程每收到新记录就 push `JSONL_RECORD`（单条）到渲染进程

### 新 IPC 通道
- `JSONL_WATCH` (invoke): `{ transcriptPath, sessionId, readFromStart }` → `{ ok }`
- `JSONL_RECORDS` (push): `{ sessionId, records: JsonlRecord[] }` （批量历史）
- `JSONL_RECORD` (push): `{ sessionId, record: JsonlRecord }` （实时单条）

```typescript
export type TimelineNodeType = 'user_input' | 'assistant' | 'tool_use' | 'tool_result'
export interface TimelineNode {
  id: string
  sessionId: string
  type: TimelineNodeType
  text?: string
  toolName?: string
  toolDisplayText?: string
  isError?: boolean
  parsedAt: number
}
export const timelineBySessionAtom = atomFamily((sessionId: string) =>
  atom<TimelineNode[]>([])
)
```

### 原子目标 6 — Claude 回复节点（T2）[x]
- `type=assistant`：白色/淡背景卡片，左侧对齐，文字内容
- **文本截断规则（PRD 3.2.1 明确要求）**：
  - 节点文字显示有字数限制，超过显示范围只显示前面部分（`white-space: nowrap; overflow: hidden; text-overflow: ellipsis`）

  - 点击节点弹出 4 选项菜单（见 T9），不是直接展开文字
- **正在执行的节点**：琥珀色闪烁动效 + `⟳` 后缀（当前正在生成 assistant 回复时）
- **「项目启动」锚点标记**：时间线最顶部的特殊节点，每个项目的历史进程的起始点（已用绿色标记实现）
