# T7 — 三种并行情形布局

## 技术原理

同一个项目下可能存在多个并发的 Claude Session，布局方式取决于它们之间的关系。

### 三种情形定义

| 情形 | 触发条件 | 布局规则 | 连线规格 |
|------|---------|---------|---------|
| **情形1 — 单主线程** | 只有一个 Session | 单框居中显示 | 无 |
| **情形2 — /branch 继承记忆** | 用户在某 Session 中输入 `/branch` | 新 Session 框并排在原 Session 右侧，框的 `top` 与 `/branch` 触发节点对齐 | 水平连线（渐变紫色，2px）+ 「继承记忆」紫色小标签 |
| **情形3 — 同项目多 Session 并行** | 同时存在多个独立 Session（非 /branch） | 所有框并排，顶部对齐 | 2px 竖向细分隔线（无标注，两侧完全独立） |

### /branch 触发识别

- Hook：`UserPromptSubmit`，payload 中 `message` 字段正则匹配 `/\/branch/`
- 识别后：
  1. 记录触发的父 Session ID + 触发节点在时间轴中的 Y 位置（像素）
  2. 新 Session 启动时（`SessionStart` Hook），检查是否有 pending branch 触发记录
  3. 有记录 → 标记新 Session 为 `/branch` 类型，关联父 Session ID

### 连线数据结构（@xyflow Edge）

```typescript
// 情形2的"继承记忆"连线（已实现，颜色已改为绿色）
interface BranchEdge extends Edge {
  type: 'branchEdge'
  source: parentSessionId       // 父框
  sourceHandle: `src-${childId}`  // 精确定位到触发节点 Y 的动态 Handle
  target: childSessionId        // 子框
  targetHandle: 'branch-target'
}
```

## 布局算法设计（当前已实现 + 待实现升级）

### Y 轴策略：时间堆叠（待实现，替代 NODE_HEIGHT_ESTIMATE 估算）

**核心思想**：新 session 的起点 Y = 其启动时刻所有已存在框的最大 bottom_y + FRAME_GAP_Y。

```
frame_bottom(id) = layout.y(id) + allFrameHeightsAtom(id)  // 框底部 Y（真实 DOM 高度）

新 session 起点 = max(所有"startedAt < 该 session.startedAt"的框的 frame_bottom) + FRAME_GAP_Y
```

- **按 startedAt 升序排序**所有 session，顺序决定 Y 堆叠顺序
- 框高来自 `allFrameHeightsAtom`（`SessionFrameNode` ResizeObserver 实测，无估算）
- 框内容增长时 `allFrameHeightsAtom` 更新 → `ProcessLineCanvas` 订阅 → 布局重算 → 后续框 Y 下移

**数据流**：
```
SessionFrameNode ResizeObserver
  → store.set(allFrameHeightsAtom, prev => new Map(prev).set(id, h))
  → ProcessLineCanvas useAtomValue(allFrameHeightsAtom) 触发重渲染
  → useSessionFrameLayout(sessionIds, heights, relations, startTimes) 重新计算
  → ReactFlow nodes 位置更新
```

**需要修改的文件**：

| 文件 | 改动 |
|------|------|
| `atoms/agent-block.atom.ts` | 新增 `allFrameHeightsAtom = atom<Map<string, number>>(new Map())` |
| `SessionFrameNode.tsx` | ResizeObserver 中改写 `allFrameHeightsAtom`（替代 atomFamily 的 `sessionFrameHeightsAtom`）|
| `ProcessLineCanvas.tsx` | `useAtomValue(allFrameHeightsAtom)` 订阅；构建 `startTimes: Map<string, number>`（从 activeSessions 取 startedAt）|
| `useSessionFrameLayout.ts` | 新增 `startTimes` 参数；Y 轴改为按 startedAt 排序后累积 `heights` 堆叠 |
| `BranchEdge.tsx` | 去掉 `targetY = sourceY` 强制水平，改为自然斜线（source Handle Y → 子框 target Handle Y）|

### X 轴策略：Cluster-aware 水平排列（已实现，保持不变）

- Branch 子框紧靠父框左/右侧（由 `SessionRelation.side` + `lineLength` 决定）
- 多个并行 Cluster 水平居中并排，间距 `FRAME_GAP_X = 40px`
- `lineLength` = `FRAME_GAP_X + (sameDirectionIndex - 1) * (FRAME_WIDTH + FRAME_GAP_X)`，使多 branch 依次向外展开

### BranchEdge source Handle Y 精确对齐（已实现）

父框 source Handle 的 `top` = `nodeYOffsetsAtom(sessionId)[triggerNodeIndex]`（DOM 实测节点底部偏移），fallback 到 `FRAME_HEADER_HEIGHT + idx * NODE_HEIGHT_ESTIMATE`。

`nodeYOffsetsAtom` 由 `SessionFrameNode` ResizeObserver 计算：
```typescript
// 查询所有可见节点（.ptl-node-user, .ptl-node-assistant），记录各节点底部相对框顶部的偏移
const offsets = Array.from(el.querySelectorAll('.ptl-node-user, .ptl-node-assistant'))
  .map(node => Math.round(node.getBoundingClientRect().bottom - el.getBoundingClientRect().top))
```

`triggerNodeIndex` 使用 `getVisibleNodeCount`（只计 user_input + assistant），与 DOM 节点序号一致。

## 已实现验收标准

- [x] 单 Session：居中显示，无连线
- [x] 触发 `/branch` 后：新 Session 框出现在右侧，有绿色（`#35C98A`）水平虚线连线 + 「继承记忆」标签，并在父框内时间轴对应位置有插入线
- [x] 两个独立 Session 并行：并排显示，无连线
- [x] 多 branch：奇数侧右、偶数侧左，依次向外展开
- [ ] **Y 轴时间堆叠对齐**：branch 子框起点与父框触发位置精确对齐（待 allFrameHeightsAtom 堆叠算法实现）
- [ ] **BranchEdge 斜线**：连线从父框 Handle 斜向子框顶部（待去掉强制水平）
