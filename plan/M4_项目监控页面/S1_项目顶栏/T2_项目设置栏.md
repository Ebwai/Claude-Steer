# T2 — 项目设置栏

## 技术原理

项目设置栏（height: 30px）显示在标签栏下方，与全局设置**完全独立**，仅影响当前项目。

### 8 个设置项 + 1 个固定按钮

PRD 要求以竖线分隔的 8 项（其中工具和经验各包含子内容），从左到右依次：

```
权限 | 模型 | Tools | MCP | CLI | Skills | 工作流 | Plugins  ↑ 同步到 GitHub
```

- 前 5 项：`设置名称 当前值(紫色) ▾`，点击展开下拉选择
- 工具 = Tools + MCP + CLI（三个子项分别独立显示）
- 经验 = Skills + 工作流（两个子项分别独立显示）
- 总共 8 个竖线分隔的条目
- **最右端固定按钮**：`↑ 同步到 GitHub`（绿色文字 + 绿色边框圆角按钮）

### 写入位置

所有设置写入当前项目的 `.claude/settings.json`（通过 `CONFIG_WRITE` IPC）。

### ↑ 同步到 GitHub 按钮

- 触发：执行当前主线版本推送（`git merge` + `git push origin`）
- 功能与进程线框底部「输入到 git 主线」效果一致，区别在于**全局触发**（不针对单节点）
- git 远程未配置或权限不足 → 弹出子窗口说明操作步骤（复用 T2_T6_Git操作实现 中的 `GitRemoteSetupModal`）

### 原子目标 2 — 8 项下拉控件 [x]

**① 权限**：6 选 1（default / acceptEdits / plan / auto / dontAsk / bypassPermissions）
- hover 每个选项显示说明文字（tooltip）
- 写入 `.claude/settings.json` 的 `permissions.defaultMode`

**② 模型**：下拉列表（Opus 4.6 / Sonnet 4.6 / Haiku 4.5 + 自定义输入）
- 写入 `model` 字段

**③ Tools**：多选 checkbox 列表（展示所有内置工具名）
- 勾选/取消勾选 → 写入 `permissions.allow`（白名单）或 `disallowedTools`

**④ MCP**：多选 checkbox 列表（展示 `~/.mcp.json` 中已配置的 MCP server 名）
- 勾选控制哪些 MCP server 在当前项目可用
- 写入 `.claude/settings.json` 的 `mcpServers` 字段

**⑤ CLI**：多选 checkbox 列表（展示所有 CLI 类 Skill 名）
- 控制哪些 CLI 工具在当前项目可用

**⑥ Skills**：多选 checkbox 列表（展示 `~/.claude/skills/` + 项目级 skills 目录中的 Skill）
- 排除 CLI 类 Skill（CLI 单独控制）

**⑦ 工作流**：多选 checkbox 列表（展示 settings.json hooks 配置中的工作流名称）
- 控制哪些 Hook 工作流在当前项目启用

**⑧ Plugins**：多选 checkbox 列表（展示 `enabledPlugins` 中的已安装 Plugin 名）
- 控制哪些 Plugin 在当前项目激活

### 原子目标 7 — ＋ 添加并行 Agent 主线按钮 [ ]

> 位置：设置栏右端，紧靠「↑ 同步到 GitHub」左侧

**按下后流程**：
1. 渲染层：调用 `IPC.SESSION_START { projectId, projectPath, permissionMode: 当前项目权限 }`
   - 与 LeftPanel「▶ 启动 Session」按钮完全相同的流程
2. 主进程：`PtyManager.startSession()` → 新开 PTY 窗口（xterm.js 子窗口）→ 执行 `claude` 命令（非 resume，全新 session）
3. 主进程：SESSION_START 成功 → 返回 `{ ok: true, sessionId: newPtyId }`
4. 渲染层：收到返回值后：
   - `activeSessionsAtom.set(newPtyId, { status: 'Running', cwd: project.path, ... })`
   - `ptySessionIdsAtom.add(newPtyId)`
5. LeftPanel 实时显示面板新增一个 AgentBlock（情形3：多 Session 并排）
6. ProcessLineCanvas 新增一个 SessionFrameNode（顶部对齐，2px 竖向分隔线）

**样式**：`color: var(--tx2); border: 1px solid var(--bd); border-radius: 4px; padding: 2px 8px; font-size: 10.5px`（紧靠 GitHub 按钮左侧，`margin-left: auto` 只加在 GitHub 按钮上）

文件：`ProjectSettingsBar.tsx`（追加按钮渲染逻辑，复用 LeftPanel 的 `handleStartSession` 逻辑）

## 验收标准
- 设置栏显示 8 个竖线分隔条目 + 最右端绿色「同步到 GitHub」按钮
- 点击任意设置项：展开对应下拉浮层
- 修改权限模式：写入项目 `.claude/settings.json`，新 Session 生效
- 点击「同步到 GitHub」：执行 merge + push，成功/失败均有视觉反馈
- 切换到不同项目 tab：设置栏自动加载该项目的配置
