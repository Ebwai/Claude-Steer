# T3 — 项目记录单持久化（projects.json）+ 后续启动重新扫描逻辑

## 技术原理

`~/.claude-driver/projects.json` 存储格式：
```json
{
  "version": 1,
  "initCompleted": true,
  "lastRootDir": "/home/tony/projects",
  "projects": [ ...Project[] ]
}
```
原子写入：write tmp → rename，防止并发写入损坏文件。

**后续启动重扫描**：app 启动时，若 `initCompleted === true` 且 `lastRootDir !== null`，
自动重扫 lastRootDir，将新发现的（不在 projects.json 中）路径以 claimStatus=0 写入——
驱动角标提示用户确认。

## 原子目标

- [x] ProjectStore.ts：  
  - `readProjectsFile()` / `writeProjectsFile(data)`：原子读写  
  - `readProjects(): Project[]`：返回所有项目  
  - `upsertProject(project)` / `updateProjectClaims(updates)`：单条/批量更新  
  - `isInitCompleted()` / `setInitCompleted(lastRootDir)`  
- [x] IPC `PROJECT_LIST`：返回 `readProjects()`（全量）
- [x] IPC `PROJECT_UPDATE`：批量 `updateProjectClaims` + 可选 `setInitCompleted`
- [x] app 启动时：loadProjects → 推送到 renderer；若 lastRootDir 存在则 auto-rescan
- [x] useIpcBridge.ts：mount 时 invoke `PROJECT_LIST` → 填充 `projectsAtom`

## 验收标准

重启 app，之前认领的项目仍在 canvas；新添加的项目目录被检测到后以角标"N 待确认"提示。
