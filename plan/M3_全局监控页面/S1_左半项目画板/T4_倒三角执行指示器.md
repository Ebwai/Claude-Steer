# T4 — 倒三角执行指示器

## 技术原理

倒三角指示器（▼）显示在项目卡片的 Plan 列表条目旁边，表示"当前正在执行该 Plan"。

### 触发条件

- **触发**：`PostToolUse` Hook 检测到 plan 文件（`plan/**/*.md`）有写入操作
- **机制**：文件变动 → `SyncPlanStatusUseCase` 重新解析 plan → 找到状态为 `[!]`（DOING）或刚被写入的最低级 plan 节点 → 在该条目旁显示倒三角

### 销毁条件

1. **正常完成**：`overall_plan.md` 中对应 M 任务全部完成（状态全为 `[x]`），且 3min 内无新 plan 文件变动 → 自动销毁
2. **可能暂停**：M 任务未完成，但 5min 内无任何 plan 文件变动 → 自动退出，显示为「可能暂停」状态（灰色文字）

### 多并行支持

- 同一个项目可同时有多个倒三角（多个 plan 并行更新时）
- 每个倒三角有独立的计时器（销毁条件 2 基于各自上次更新时间）

## 实现步骤

### 原子目标 1 — PlanIndicator 数据结构 [x]
文件：`src/shared/types/planIndicator.ts`
```typescript
interface PlanIndicator {
  planNodeId: string        // 对应 plan 节点的 ID（如 'M1-S2-T1'）
  projectId: string
  status: 'active' | 'possibly-paused' | 'completed'
  lastUpdatedAt: number     // 最后一次 plan 文件变动时间戳
  triggeredAt: number       // 首次出现时间戳
}
```

### 原子目标 2 — SM5/SM6 状态管理服务 [x]
文件：`src/main/services/PlanSyncService.ts`（扩展）
- 订阅 `PostToolUse` Hook，工具名为 `Write/Edit`，判断 `tool_input.file_path` 是否匹配 `plan/**/*.md`
- 匹配 → 读取并解析修改的 plan 文件 → 找出 DOING 状态节点 → 广播 `ShowIndicatorEvent`
- 定时器管理：
  - 每个活跃指示器启动独立 5min 倒计时
  - 收到同一 plan 的新变动时：重置倒计时
  - 5min 到期：状态改为 `possibly-paused`，通知渲染进程
  - M 任务全完成时：3min 倒计时后销毁

### 原子目标 3 — planIndicatorsAtom [x]
文件：`src/renderer/src/atoms/projects.atom.ts`（追加）
```typescript
export const planIndicatorsByProjectAtom = atomFamily((projectId: string) =>

```

### 原子目标 4 — 项目卡片渲染更新 [x]
文件：`src/renderer/src/features/global-monitor/ProjectCard.tsx`（扩展）
- 订阅 `planIndicatorsByProjectAtom(projectId)`
- 在对应 plan 条目旁渲染倒三角：
  ```jsx
  <span className="plan-indicator active">▼</span>   // 绿色
  <span className="plan-indicator paused">▼</span>   // 灰色（可能暂停）
  ```
- 同一条目可有多个倒三角（多并行情形）

## 新增 IPC 通道

- `PLAN_INDICATOR_UPDATE` (push): `{ projectId, indicators: PlanIndicator[] }` — 指示器状态更新

## 验收标准

- Claude 修改 plan 文件时，对应 plan 条目旁出现绿色倒三角动效
- 5min 无 plan 变动 → 倒三角变灰 + 「可能暂停」文字
- M 任务全完成后 3min → 倒三角消失
- 多个 plan 同时被修改 → 多个倒三角同时存在
