# T1-T4 — 全局设置 Modal 实现（合并）

## 技术原理

全局设置以居中 Modal 浮窗方式呈现（非独立页面），读写 `~/.claude/settings.json` 和 `~/.claude-driver/config.json`。

### Modal 规格

- 宽 640px，最大高度 80vh
- 顶部固定锚点横向导航（10 个分区 Tab）+ 下方滚动内容区（`overflow-y: auto`，`max-height: calc(80vh - 100px)`）
- 复用现有通用 `Modal` 组件（`width={640}`），不修改 Modal 本体
- 背景遮罩由 Modal 组件自带（backdrop-filter blur(2px)）
- 右上角 ✕ 按钮 + 点击遮罩关闭（Modal 组件已实现）

> ⚠️ **注意**：Plan 原设计为左侧纵向锚点导航，但 Modal 640px 宽度内再分左右栏空间极紧张。
> 改为**顶部横向 Tab 锚点**（10 个 Tab，flex-wrap 两行），下方单列滚动。
> 与 `InitSopModal` 的步骤导航风格一致，复用已有 CSS 变量体系。

### 配置文件对应关系

| 设置分区 | 写入位置 | scope 参数 |
|---------|---------|------------|
| API/认证 | `~/.claude/settings.json` | `claude` |
| 模型/推理 | `~/.claude/settings.json` | `claude` |
| 语言 | `~/.claude/settings.json` | `claude` |
| 权限 | `~/.claude/settings.json` | `claude` |
| Token/费用 | `~/.claude-driver/config.json` | `driver` |
| 通知 | `~/.claude-driver/config.json` | `driver` |
| 偏好 | `~/.claude/settings.json` + `~/.claude-driver/config.json` | 按字段分别写 |
| 记忆 | `~/.claude/settings.json` | `claude` |
| 存储 | `~/.claude/settings.json` + `~/.claude-driver/config.json` | 按字段分别写 |
| 关于 | 只读 | — |

### DriverConfig 数据结构（新增）

```typescript
// shared/types/index.ts 追加
interface DriverConfig {
  tokenPriceInputPerM: number      // 输入 token 单价（$/百万），默认 3.0
  tokenPriceOutputPerM: number     // 输出 token 单价（$/百万），默认 15.0
  monthlyBudgetAlertUsd: number    // 月度预算警告阈值（$），默认 50
  desktopNotificationsEnabled: boolean  // 桌面通知总开关，默认 true
  themePreference: 'dark' | 'light'     // 主题偏好，默认 'dark'
}
```

### 原子目标 T4 — 配置导出/导入 + API 连通性测试 + 关于分区 [x]

**导出配置**：
- 按钮调用 `IPC.DIALOG_SAVE_FILE`（新增）→ Electron `dialog.showSaveDialog` → 返回路径
- 再调用 `IPC.CONFIG_EXPORT { targetPath }` 将 `~/.claude-driver/config.json` 复制到目标路径

**导入配置**：
- 按钮调用 `IPC.DIALOG_OPEN_FILE`（新增）→ `dialog.showOpenDialog`，过滤 `.json`
- 调用 `IPC.CONFIG_IMPORT { sourcePath }` 读取、校验、写入 `~/.claude-driver/config.json`
- 成功后重新调用 `DRIVER_CONFIG_READ` 刷新所有设置分区

**API 连通性测试**：
- 调用 `IPC.API_TEST { apiKey }` → 主进程发极短 messages 请求（max_tokens=1）
- 成功 → 绿色「连通性正常」+ 模型名；失败 → 红色错误信息

**关于分区**：
- 当前版本号（从 `package.json` 读取，经由 `app.getVersion()`，加到 `DRIVER_CONFIG_READ` 响应中，或单独一个 `APP_VERSION_READ` invoke）
- 「检查更新」→ `shell.openExternal` 打开 GitHub Releases 页
- 开源地址链接

## 新增/复用 IPC 通道（修订版）

| 通道常量 | 方向 | payload | 说明 |
|---------|------|---------|------|
| `DRIVER_CONFIG_READ` | invoke | — | 读取 `~/.claude-driver/config.json` → 返回 `DriverConfig` + `appVersion` |
| `CONFIG_WRITE` （已有，补实现）| invoke | `{ scope: 'claude' \| 'driver', key: string, value: unknown }` | 按 scope 写入对应配置文件（原子写入） |
| `API_TEST` | invoke | `{ apiKey: string }` | 测试 API Key 连通性 → `{ ok, model? } \| { error }` |
| `CONFIG_EXPORT` | invoke | `{ targetPath: string }` | 复制 config.json → `{ ok }` |
| `CONFIG_IMPORT` | invoke | `{ sourcePath: string }` | 读取+校验+写入 config.json → `{ ok } \| { error }` |
| `DIALOG_SAVE_FILE` | invoke | `{ defaultName?: string }` | 弹出保存对话框 → 路径或 null |
| `DIALOG_OPEN_FILE` | invoke | `{ filters?: FileFilter[] }` | 弹出打开文件对话框 → 路径或 null |

**删除的原计划通道（不再新增）**：
- ~~`SETTINGS_READ`~~ → 拆分为 `DRIVER_CONFIG_READ`（driver 侧）+ 现有 `CONFIG_READ` 补字段
- ~~`SETTINGS_WRITE`~~ → 复用并实现现有 `CONFIG_WRITE`
- ~~`THEME_CHANGE`~~ → renderer 直接改 CSS 变量，不需要主进程
- ~~`CHECK_UPDATE`~~ → `shell.openExternal` 打开 GitHub 页，不需要主进程 IPC

## 新增主进程模块

### `src/main/lib/config/DriverConfigStore.ts`（新建）

与 `ProjectStore.ts` 同级，遵循相同的原子写入模式：
- `readDriverConfig(): DriverConfig`
- `writeDriverConfig(data: DriverConfig): void`（write-tmp + rename）
- `patchDriverConfig(key: keyof DriverConfig, value: unknown): void`
- 路径：`~/.claude-driver/config.json`（与 `projects.json` 同目录）

> ⚠️ IPC handler 中禁止直接操作文件，必须通过 `DriverConfigStore` 委托

## 废弃计划

原 T1-T4 IPC 通道设计（2026-04-14 版本）中以下通道不再新增：

- `SETTINGS_READ` — 原因：与现有 `CONFIG_READ` 语义重叠，driver 配置用新增 `DRIVER_CONFIG_READ` 覆盖
- `SETTINGS_WRITE` — 原因：`CONFIG_WRITE` stub 已存在，直接补实现即可，避免功能重复
- `THEME_CHANGE` — 原因：主题切换是纯 renderer 行为（修改 CSS 变量），不需要跨进程
- `CHECK_UPDATE` — 原因：只需打开浏览器到 GitHub Releases，`shell.openExternal` 足够，无需额外 IPC
