# T1-T3 — 远程交互（cc-connect）实现（合并）

## 技术原理

远程交互基于 **cc-connect** 开源项目，实现飞书 IM 与 Claude Code 之间的桥接。
架构文档详见 `架构.md §4.11`。

### cc-connect 配置文件格式（`~/.cc-connect/config.toml`）

cc-connect 采用 `[[projects]]` 多项目嵌套 TOML 格式，**一个进程管理所有项目**：

```toml
[[projects]]
name = "项目A名称"
work_dir = "/path/to/projectA"

  [[projects.bots]]
  platform = "feishu"
  agent = "claude-code"
  enable_feishu_card = true
  progress_style = "card"

    [projects.bots.feishu_config]
    app_id = "cli_xxx"
    app_secret = "xxx"
    admin_from = "ou_xxx"

  [[projects.bots.allowed_senders]]
  platform = "feishu"
  id = "ou_xxx"
```

**注意**：旧 plan 中的扁平 TOML 格式（`work_dir = "..."` 顶层直写）是错误的，cc-connect 实际使用 `[[projects]]` 嵌套结构。

### 飞书 Bot 配置数据存储

存入 `~/.claude-driver/projects.json` 的 `project.feishuBot` 字段（`FeishuBotConfig` 类型）：

```typescript
interface FeishuBotConfig {
  appId: string
  appSecret: string
  adminFrom: string
  allowedSenders: string[]
  enableFeishuCard: boolean
  progressStyle: 'legacy' | 'compact' | 'card'
}
```

### 触发入口

点击「📡 远程交互」→ 打开 `RemoteModal`（与 `SoulModal`、`SchedulerModal` 同级）。

## 实现步骤

### 原子目标 T1 — cc-connect 安装检测 + 引导安装 [x]

**检测安装（主进程）**：
- `IPC.CC_CONNECT_CHECK`：`execFile('which', ['cc-connect'])` (Unix) / `execFile('where', ['cc-connect'])` (Windows)
- **不开 PTY**，同步 execFile 执行，返回 `{ installed, version? }`

**UI（RemotePanel）**：
- 已安装 → 显示版本号 + 服务状态区
- 未安装 → 显示灰色「尚未安装 cc-connect」提示 + 「一键安装」按钮
  - 点击「一键安装」→ 调用 `CHAT_START + CHAT_WINDOW_OPEN`（复用闲聊 PTY）→ 打开交互终端窗口，窗口内自动写入 `npm install -g cc-connect\r`（npm 是 cc-connect 推荐安装方式）

文件：`src/renderer/src/features/remote/RemotePanel.tsx`

### 原子目标 T2 — 飞书平台配置向导（5步）[x]

文件：`src/renderer/src/features/remote/FeishuConfigWizard.tsx`

- **步骤进度条**（Step 1/5 ~ 5/5）
- **Step 1**：创建飞书自建应用 — 图文说明 + 「打开飞书开发者控制台」链接按钮（`https://open.feishu.cn/app`）
- **Step 2**：配置应用权限 — 列出所需 8 项权限（`im:message`、`im:message.group_msg`、`im:message.p2p_msg:readonly`、`im:message.reactions:read`、`im:resource`、`im:chat`、`contact:user.base:readonly`、`contact:user.id:readonly`）
- **Step 3**：配置长连接事件订阅 — 「事件与回调」→「订阅方式」→ 选「**使用长连接接收事件**」（非 Webhook）→ 添加事件 `im.message.receive_v1`（**不是 Webhook URL**）
- **Step 4**：填写凭证表单
  - `app_id` / `app_secret`（输入框）
  - `admin_from`（open_id 输入框 + 说明：先启动服务，发一条消息，查看日志中的 open_id）
  - `allowed_senders`（标签式列表输入，回车/空格确认一个）
  - `enable_feishu_card`（开关，默认 true）
  - `progress_style`（下拉：legacy / compact / card，默认 card）
- **Step 5**：发布应用 — 「版本管理与发布」→「创建版本」→ 申请发布 → 管理员审批 → 在飞书 App 中搜索机器人添加

### 原子目标 T3 — 配置文件生成 + 服务启停 + 状态检测 [x]

文件：`src/main/services/RemoteBridgeService.ts`

**配置文件生成**：
- 从 `projects.json` 读取所有 `feishuBot` 非空的项目 → 生成正确 TOML 嵌套格式 → 写入 `~/.cc-connect/config.toml`
- 创建目录如不存在（`~/.cc-connect/`）
- 每次 `CC_CONNECT_CONFIG_SAVE` 后自动重新生成并重启服务

**服务启动**：
- `spawn('cc-connect', ['start'])` （cc-connect 默认读取 `~/.cc-connect/config.toml`）
- 捕获 stdout/stderr → 通过 `CC_CONNECT_LOG` IPC push 推送日志（最新 50 行）

**服务停止**：向子进程发 SIGTERM，等待退出

**状态检测**：每 5s 检测进程是否存活（`process.kill(pid, 0)`），状态变更时推送 `CC_CONNECT_STATUS`

## 新增 IPC 通道

| 通道 | 方向 | 签名 |
|------|------|------|
| `CC_CONNECT_CHECK` | invoke | `{}` → `{ installed: boolean, version?: string }` |
| `CC_CONNECT_START` | invoke | `{}` → `{ ok: boolean }` |
| `CC_CONNECT_STOP` | invoke | `{}` → `{ ok: boolean }` |
| `CC_CONNECT_STATUS` | invoke | `{}` → `{ running: boolean }` |
| `CC_CONNECT_CONFIG_SAVE` | invoke | `{ projectId: string, bot: FeishuBotConfig }` → `{ ok: boolean }` |
| `CC_CONNECT_LOG` | push | `{ log: string }` — 实时日志推送 |
