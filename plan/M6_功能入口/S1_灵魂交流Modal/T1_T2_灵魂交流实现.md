# T1-T2 — 灵魂交流 Modal 实现（合并）

## 技术原理

灵魂交流 Modal 是从全局监控右半「功能入口」区块触发的轻量浮窗，
提供两个功能卡片：Auto-Dream（暂未开放）和 /insight 使用洞见。

### Modal 规格

- 半透明黑色遮罩 + `backdrop-filter: blur(2px)`
- 圆角卡片：宽 360px，居中显示
- 头部：`💫 灵魂交流` + ✕ 关闭按钮
- 两个功能卡片垂直排列，hover 时边框轻微高亮
- 点击遮罩 / ✕ 关闭

## 实现步骤

### 原子目标 T1 — Auto-Dream 卡片 [x]
文件：`src/renderer/src/features/modals/SoulCommunionModal.tsx`

**卡片内容**：
- 图标：`✧ Auto-Dream`
- 说明文字：「Claude Code 内置后台记忆整合引擎，会话结束后自动提炼工作习惯与项目约定写入持久记忆。当前版本尚未全面开放。」
- **点击卡片 → 弹出内层 Modal**：「此功能 Claude Code 暂时未全面开放，敬请期待」
- **重要约束**：不写入任何 `settings.json` 字段（`autoDreamEnabled` 字段不读写）

### 原子目标 T2 — /insight 卡片 [x]

**卡片内容**：
- 图标：`💡 /insight 使用洞见`
- 说明文字：「分析 token 用量、项目进展、使用习惯，生成可视化 HTML 报告」
- **点击后流程**：
  1. 卡片变为转圈等待状态（`⟳ 生成中...`）
  2. 向当前活跃 Session 的 PTY stdin 写入 `/insights`（**注意复数形式**）
  3. 监听 `~/.claude/usage-data/report.html` 文件是否创建（`chokidar` 监听，超时 60s）
  4. 文件出现 → 转圈位置替换为「📂 打开报告」按钮
  5. 点击「打开报告」→ 在 Electron 内嵌 WebView 中打开（**不使用系统浏览器**）

**/insights 发送目标 Session**：
- 优先使用当前项目监控页面中选中的 Session
- 无选中 Session → 使用最近活跃的 Session
- 无任何活跃 Session → 提示「请先启动一个 Claude Session」

**WebView 实现**：
```typescript
// 在 Electron Main Process 创建 BrowserWindow 或使用 Renderer 中的 <webview> 标签
// 推荐：独立的 BrowserWindow（height: 80vh, width: 900px）
mainWindow.webContents.openDevTools // 不需要
new BrowserWindow({ file: reportHtmlPath })
```

### 原子目标 T2.1 — 报告文件检测 IPC [x]
- `INSIGHT_REPORT_WATCH` (invoke): `{}` → 开始监听 `~/.claude/usage-data/report.html`
- `INSIGHT_REPORT_READY` (push): `{ filePath }` → 文件就绪通知

### 原子目标 T2.2 — WebView 打开 [x]
- `OPEN_WEBVIEW` (invoke): `{ filePath }` → 在新 BrowserWindow 中打开 HTML 文件
  - `shell.openExternal` 方案（系统浏览器）被明确排除，必须用内嵌 WebView

## 验收标准

- 点击 Auto-Dream 卡片：弹出「尚未全面开放」内层 Modal，不修改任何配置
- 点击 /insight 卡片：显示转圈，向 Claude 发送 `/insights`
- 报告生成后：转圈替换为「打开报告」按钮
- 点击「打开报告」：在应用内新窗口（WebView）打开 HTML 报告
