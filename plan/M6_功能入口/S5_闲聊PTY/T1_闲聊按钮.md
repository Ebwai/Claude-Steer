# T1 — 💬 闲聊（聊天气泡式 UI）

## 技术选型与原理

### 为什么不用 TERM_WINDOW_OPEN + TerminalPage
`TerminalPage` 使用 xterm.js 渲染原始 PTY 字节流（ANSI 转义序列），是终端模拟器，不是聊天界面。用户看到的是 Claude CLI 的完整终端输出，包括提示符、光标等噪音，不符合"聊天软件"体验。

### 核心方案：stream-json 输出 + 独立 ChatPage

`claude --output-format stream-json` 会让 Claude CLI 以 NDJSON 格式输出，每行一个 JSON 对象（`type: assistant`/`system`/`result` 等），主进程可解析后推送结构化消息，renderer 渲染气泡。

流程：
1. `CHAT_START`：主进程 `startBare` 传入 `['--output-format', 'stream-json', '--dangerously-skip-permissions']`
2. `onData` 回调：buffer 累积跨行 JSON，解析后推 `CHAT_MESSAGE` IPC 到 chat 窗口
3. `TERM_WINDOW_OPEN` 替换为新的 `CHAT_WINDOW_OPEN`，加载 `#/chat?sessionId=id`
4. `ChatPage.tsx`：订阅 `CHAT_MESSAGE`，渲染气泡，底部输入框发 `SESSION_INPUT`

### A2 — 主进程：修改 CHAT_START handler + 新增 CHAT_WINDOW_OPEN

**文件**：`src/main/index.ts`

`CHAT_START`：改为用 `startBare` 传 `stream-json` args，`onData` 解析 JSON 推送 `CHAT_MESSAGE`：

```typescript
// chatWindows: Map<chatId, BrowserWindow>  — 在 registerIpcHandlers 内声明
const chatWindows = new Map<string, BrowserWindow>()

ipcMain.handle(IPC.CHAT_START, async () => {
  const chatId = `chat-${randomUUID()}`
  const homeDir = os.homedir()
  chatPtyIds.add(chatId)

  let buf = ''  // 跨行 buffer，处理 JSON 被分片推送的情况

  ptyManager.startBare(
    {
      sessionId: chatId,
      projectPath: homeDir,
      permissionMode: 'default',
      onData: (_id, raw) => {
        buf += stripAnsi(raw)
        const lines = buf.split('\n')
        buf = lines.pop() ?? ''   // 末尾未完整行留给下次
        for (const line of lines) {
          const trimmed = line.trim()
          if (!trimmed) continue
          try {
            const obj = JSON.parse(trimmed) as Record<string, unknown>
            chatWindows.get(chatId)?.webContents.send(IPC.CHAT_MESSAGE, obj)
          } catch { /* 非 JSON 行（提示符等）忽略 */ }
        }
      },
      onExit: (sid, code) => {
        console.log(`[chat] PTY exited (id=${sid} code=${code})`)
        chatPtyIds.delete(sid)
        chatWindows.get(sid)?.webContents.send(IPC.CHAT_MESSAGE, { type: 'system', subtype: 'exit' })
      },
    },
    ['--output-format', 'stream-json', '--dangerously-skip-permissions']
  )
  return { ok: true, sessionId: chatId }
})

ipcMain.handle(IPC.CHAT_WINDOW_OPEN, async (_event, payload: { sessionId: string }) => {
  const { sessionId } = payload
  const existing = chatWindows.get(sessionId)
  if (existing && !existing.isDestroyed()) { existing.focus(); return { ok: true } }

  const win = new BrowserWindow({
    width: 640, height: 760, minWidth: 480, minHeight: 400,
    title: '💬 闲聊 — Claude',
    webPreferences: { preload: join(__dirname, '../preload/index.js'), contextIsolation: true },
  })
  chatWindows.set(sessionId, win)
  win.on('closed', () => chatWindows.delete(sessionId))

  if (is.dev && process.env['ELECTRON_RENDERER_URL']) {
    await win.loadURL(`${process.env['ELECTRON_RENDERER_URL']}#/chat?sessionId=${encodeURIComponent(sessionId)}`)
  } else {
    await win.loadFile(join(__dirname, '../renderer/index.html'), {
      hash: `/chat?sessionId=${encodeURIComponent(sessionId)}`,
    })
  }
  return { ok: true }
})
```

### A5 — ChatPage.tsx + ChatPage.css（新建）

**文件**：`src/renderer/src/features/chat/ChatPage.tsx`

```
消息结构（来自 stream-json）：
  type: 'assistant'  → content[].type='text' → 气泡（左侧，Claude 配色）
  type: 'system' subtype: 'init' → 忽略
  type: 'system' subtype: 'exit' → 显示"对话已结束"提示
  type: 'result' → 可选：显示用量摘要
```

UI 结构：
- 顶部标题栏（「💬 闲聊 — Claude」+ 关闭按钮）
- 中间消息列表（flex-col, overflow-y scroll, 新消息自动滚到底）
  - 用户消息：右侧蓝色气泡
  - Claude 消息：左侧深色气泡，支持 markdown 渲染（复用现有 marked 或直接 pre/whitespace-pre-wrap）
- 底部输入框（textarea, Shift+Enter 换行, Enter 发送, 发送后清空）
  - 发送 → `window.api.invoke(IPC.SESSION_INPUT, { sessionId, text: input })`

## 验收标准

1. 点击「💬 闲聊」→ 弹出 640×760 聊天窗口
2. 窗口底部输入框可输入，Enter 发送
3. Claude 回复以气泡形式出现在左侧
4. 用户消息以气泡形式出现在右侧
5. 关闭窗口后 PTY 被清理
6. 全局监控画板、项目列表、进程线无任何副作用
