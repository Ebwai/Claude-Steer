# T3 — 建立共享类型层

## 技术原理

**跨进程类型共享的问题**：
- electron-vite 将 main 进程编译为 Node.js ESM/CJS，renderer 编译为 browser bundle（Vite 处理）
- 两个进程共用相同的 TypeScript 类型定义，但这些类型**不包含运行时逻辑**（纯 `.ts` interface/type/const）
- 放置位置：`src/renderer/src/shared/`，通过 tsconfig.web.json 的 `@renderer` alias 可在 renderer 中导入
- main 进程中使用这些类型时，**通过相对路径** import，或直接在 main 层重新声明等价类型（两者都合理，MVP 阶段统一用相对路径）

**三个子目录职责**：
- `types/` — Domain 实体接口（Project、Session、PlanNode 等）
- `events/` — IPC 通道名常量（`ipc-channels.ts`）
- `constants/` — 跨进程常量（端口号、路径前缀等）
