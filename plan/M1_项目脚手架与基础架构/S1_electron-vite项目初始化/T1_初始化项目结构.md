# T1 — 初始化 electron-vite + React + TypeScript 项目

## 技术原理与选择理由

**electron-vite** 是基于 Vite 的 Electron 构建工具，核心优势：
- **三目标独立构建**：main（Node.js）/ preload（受限 Node.js）/ renderer（Chromium）各自独立 Vite 配置，互不干扰
- **HMR 真正有效**：renderer 享有 Vite 的毫秒级热更新；main 进程变更触发 Electron 重启（而非全量重编译）
- **native module 友好**：通过 `externalizeDepsPlugin` 控制哪些包不被 bundle，主进程 native addon（node-pty）走 require 加载

官方模板 `react-ts`：预配置 React 18 + TypeScript 5 + `@vitejs/plugin-react`，开箱即用。

## 原子步骤

### 步骤 1：脚手架生成
```bash
cd /home/tony/CLAUDE_Steer
npm create @quick-start/electron@latest claude-driver -- --template react-ts
```
> 预期结果：生成 `claude-driver/` 目录，包含 electron-vite 标准结构

### 步骤 2：安装依赖
```bash
cd /home/tony/CLAUDE_Steer/claude-driver
npm install
```

### 步骤 3：验证项目可启动
```bash
cd /home/tony/CLAUDE_Steer/claude-driver
npm run dev
```
> 预期结果：Electron 窗口打开，显示 Vite + React 默认页面，无报错

### 步骤 4：整理目录结构（对照架构.md §5）

生成的标准结构为：
```
claude-driver/
├── electron/
│   ├── main/index.ts
│   └── preload/index.ts
├── src/
│   ├── App.tsx
│   └── main.tsx
├── electron-vite.config.ts
├── package.json
└── tsconfig.json
```

需要**额外创建**的目录（供 T2-T4 使用）：
```bash
mkdir -p src/shared/types src/shared/events src/shared/constants
mkdir -p src/atoms src/hooks src/components src/features src/styles
mkdir -p electron/main/ipc electron/main/services electron/main/lib
```

### 步骤 5：更新 package.json name 字段
确认 `package.json` 中：
- `"name": "claude-driver"`
- `"version": "0.1.0"`
