## 2. 多平台构建流水线

### 2.1 Workflow 结构

```
.github/workflows/build.yml
```

```
触发条件:
  push → main       → 三平台构建 + 发布到 GitHub Releases
  PR   → main       → 三平台构建（仅验证编译通过性）
  tag  → v*         → 正式发版
  workflow_dispatch  → 手动按需触发

构建矩阵:
  ubuntu-latest  → .AppImage
  macos-latest   → .dmg
  windows-latest → .exe (NSIS)

产物:
  每个平台一个 artifact，可在 Actions 页面下载
```

### 2.2 构建流程（每个平台）

```
1. checkout 代码
2. setup Node.js 22（缓存 npm）
3. npm ci                    → 安装依赖（含 postinstall: electron-builder install-app-deps）
4. npx electron-builder install-app-deps  → 显式重编原生模块（node-pty）
5. npm run build:${target}   → typecheck + electron-vite build + electron-builder
6. upload-artifact           → 上传 .dmg / .AppImage / .exe
```

### 2.3 GitHub Releases 更新链

```
开发者 push tag v0.2.0
  ↓
GitHub Actions 三平台构建
  ↓
electron-builder 自动上传产物到 GitHub Release
  ↓
已安装的旧版本应用内 electron-updater 检测到新版本
  ↓
用户收到更新通知 → 一键下载安装
```

electron-builder.yml 中的 publish 配置：
```yaml
publish:
  provider: github
```

这意味着每次 main 分支 push 或 tag push 时，构建产物会自动关联到 GitHub Release。

## 3. macOS VNC 测试方案

### 3.1 为什么需要 VNC

GitHub Actions macOS runner 自带 Window Server，Electron 应用可以启动并显示 GUI。但没有内建 VNC，你需要通过社区方案打隧道连进去，**像操作一台远程 Mac 一样手动测试你的应用**。

### 3.2 推荐方案：fastmac-VNCgui

选用 [fastmac-VNCgui](https://github.com/CONIGUERO/fastmac-VNCgui) 项目——专门为 GitHub Actions macOS runner 设计的 VNC 方案。

**原理**：
```
GitHub Actions macOS Runner
  ├── 内置 VNC Server（macOS Screen Sharing）
  │     └── 端口 5900，仅监听 127.0.0.1
  ├── ngrok tcp 隧道
  │     └── 将 127.0.0.1:5900 暴露为公网 tcp://x.tcp.ngrok.io:xxxxx
  └── 你的 Electron 应用
        └── npm run dev 启动

你的本地电脑
  └── VNC 客户端 → 连接 ngrok 公网地址 → 看到 macOS 桌面 → 操作应用
```

### 3.3 新建 VNC 测试 Workflow

在 `.github/workflows/` 下新建 `macos-vnc-test.yml`：

```yaml
name: macOS VNC Interactive Test

on:
  workflow_dispatch: {}

jobs:
  vnc:
    runs-on: macos-latest
    timeout-minutes: 60

    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: npm
          cache-dependency-path: claude-driver/package-lock.json

      - name: Install dependencies
        working-directory: claude-driver
        run: npm ci

      - name: Install Claude Code CLI
        run: npm install -g @anthropic-ai/claude-code

      # ── 开启 macOS 内置 VNC Server ──────────────────────────────────
      - name: Enable VNC
        run: |
          sudo /System/Library/CoreServices/RemoteManagement/ARDAgent.app/Contents/Resources/kickstart \
            -activate -configure -access -on \
            -clientopts -setvnclegacy -vnclegacy yes \
            -clientopts -setvncpw -vncpw "test1234" \
            -restart -agent -privs -all
          # 确认 VNC 端口已监听
          sudo lsof -i :5900 || echo "VNC port check"

      # ── ngrok 隧道（需要 NGROK_AUTH_TOKEN secret）─────────────────
      - name: Start ngrok tunnel
        uses: ngrok/ngrok-action@v1
        with:
          authtoken: ${{ secrets.NGROK_AUTH_TOKEN }}
          protocol: tcp
          port: 5900

      - name: Start Electron app
        working-directory: claude-driver
        run: npm run dev &
        env:
          GH_TOKEN: ${{ secrets.GITHUB_TOKEN }}

      # ── 等待用户手动测试（最长 50 分钟）───────────────────────────
      - name: Wait for manual testing
        run: |
          echo "=============================================="
          echo "  VNC 连接信息："
          echo "  地址：检查上方 ngrok 输出的 tcp 隧道地址"
          echo "  端口：ngrok 分配的端口"
          echo "  密码：test1234"
          echo ""
          echo "  连接后可在桌面 Dock 中找到 claude-driver"
          echo "  本 session 最长持续 50 分钟"
          echo "=============================================="
          sleep 3000  # 50 分钟
```

### 3.4 使用步骤

**一次性准备**：
1. 注册 [ngrok](https://ngrok.com/) 账号（免费版即可）
2. 获取 authtoken
3. 在 GitHub 仓库 → Settings → Secrets → 添加 `NGROK_AUTH_TOKEN`

**每次测试**：
1. 进入 Actions → `macOS VNC Interactive Test` → Run workflow
2. 等待约 2-3 分钟（安装依赖 + 启动）
3. 在 workflow 日志中找到 VNC 连接地址
4. 本地用 VNC 客户端（macOS 自带：`Cmd+Space` → "屏幕共享"）连接
5. 看到 macOS 桌面后，从 Dock 启动 claude-driver
6. 手动验证 GUI 功能
7. 完成后手动 cancel workflow 释放资源

**免费版 ngrok 限制**：
- 并发连接数有限
- 带宽受限，GUI 可能略有延迟
- 不影响功能测试

## 4. 发布流程

### 4.1 版本号规范

遵循 SemVer：`v<major>.<minor>.<patch>`

当前版本 `0.1.0`（package.json），建议首次多平台发布用 `v0.2.0`。

### 4.2 发版步骤

```bash
# 1. 确保所有改动已提交、PR 已合并到 main
git checkout main
git pull

# 2. 更新版本号
cd claude-driver
npm version minor    # 0.1.0 → 0.2.0，自动创建 tag 和 commit

# 3. 推送 tag
git push origin v0.2.0
git push origin main

# 4. 等待 CI 完成（约 5-10 分钟）
#    https://github.com/Ebwai/Claude_Steer_Test/actions

# 5. 检查 GitHub Releases
#    https://github.com/Ebwai/Claude_Steer_Test/releases
#    CI 会自动上传 .dmg / .AppImage / .exe 到对应 Release

# 6. 手动编辑 Release 说明，补充：
#    - 各平台安装说明
#    - macOS 已知问题（见下方）
#    - 测试验证结果
```

### 4.3 macOS 已知问题声明

在 Release Notes 中包含以下内容：

```markdown
### macOS 已知限制

- **系统通知**：未签名的应用在 macOS 上无法发送桌面通知。
  此功能在签名公证之前不可用。不影响其他所有功能。
- **首次打开**：由于应用未公证，首次打开时需要右键点击
  claude-driver.app → "打开" 来绕过 Gatekeeper。
- **PATH 环境**：从 Dock/Launchpad 启动时，应用会自动尝试
  恢复 PATH 以找到 claude 和 git 命令。如果遇到 "command not found"
  错误，请从 Terminal 启动应用：
  ```bash
  open /Applications/claude-driver.app
  ```
```

## 5. 验证清单

### 发版前手动验证（macOS VNC）

| 检查项 | 方法 |
|--------|------|
| 应用启动正常 | 从 Dock 启动，窗口正常渲染 |
| PTY session 启动 | 选一个项目，点 Start，查看终端输出 |
| Hook 系统工作 | 查看 Hook Server 日志（`~/.claude-steer/logs/latest.log`） |
| statusLine 桥接 | 查看终端中是否出现 token 统计 |
| JSONL 实时监听 | 查看时间线是否有新节点出现 |
| Git 操作 | 测试 commit / reset 功能 |
| 窗口操作 | 最大化/最小化/全屏/关闭 |
| 独立终端窗口 | 打开终端子窗口，输入命令 |

### 发版前自动验证（CI）

| 检查项 | 方法 |
|--------|------|
| 三平台编译通过 | CI build job 全部绿 |
| TypeScript 类型检查 | `npm run build` 包含 typecheck |
| 单元测试 | `npm test`（需要在 CI 中增加 test job） |
| 安装包生成 | CI artifacts 中包含正确命名的安装包 |
